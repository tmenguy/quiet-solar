### [nice-to-have] Replace the `": No forecast"` string-coupling with a shared sentinel constant
- File: `custom_components/quiet_solar/ha_model/car.py:1170` (and the sentinel
  source `ha_model/person.py:~342` / the join in `car.py:~417`)
- Severity: nice-to-have
- Source: blind-hunter + edge-case-hunter
- Description: `get_car_charge_origin_readable_string` suppresses the empty
  forecast via `forecast.endswith(": No forecast")`, a literal-string coupling
  to the sentinel emitted by `get_forecast_readable_string()`. If the sentinel
  text or the `"<name>: <forecast>"` join format ever changes, the suffix
  check silently stops matching and the card reverts to showing
  `"Forecasted from <name>: No forecast"`.
- Proposed fix: Promote the `"No forecast"` sentinel to a shared module-level
  constant (e.g. in `const.py` or alongside `get_forecast_readable_string`),
  use it both where the sentinel is produced and where it is matched. Prefer a
  structured check (mileage/leave-time is None) if it is reachable without
  re-deriving the forecast. Keep the existing test
  `test_empty_forecast_person_origin` green.

### [nice-to-have] Handle the far-out (>~24h) two-line date in the origin string
- File: `custom_components/quiet_solar/ha_model/car.py:1176,1180` (Calendar /
  Manual branches; formatter at `home_model/constraints.py:~73`)
- Severity: nice-to-have
- Source: edge-case-hunter
- Description: For a Calendar/Manual target more than ~24h out,
  `get_readable_next_target_date_string(for_small_standalone=True)` returns a
  two-line `"%m-%d\n%H:%M"` form, so the origin row renders
  `"Calendar · 06-26\n14:30"`. HTML collapses the newline to a space, but the
  `·`-plus-two-part-date reads awkwardly in the single-line context row.
- Proposed fix: Normalize the embedded newline for the single-line context row
  — e.g. replace `\n` with a space (or `" · "`) when composing the
  `"Calendar · {target}"` / `"Manually set to {target}"` strings — so the
  far-out case renders cleanly on one line. Add/extend a test in
  `tests/test_car_charge_origin.py` with a >24h `end_of_constraint` asserting
  the rendered string contains no raw newline.
