## Findings to fix

### [must-fix] #1 — ECG trace invisible on light themes (mix-blend-mode washout)
- File: `custom_components/quiet_solar/ui/resources/qs-car-card.js:726`
- Severity: must-fix (user-confirmed visual regression)
- Source: qs-review-blind-hunter + qs-review-edge-case-hunter + user repro
- Description: The new ECG wrapper `<g clip-path="url(#${ecgClipId})"
  pointer-events="none" style="mix-blend-mode:screen;">` uses screen
  blend, which brightens against dark backgrounds but effectively
  erases against light ones. On a light HA theme the flatline cyan→blue
  trace becomes invisible. The pre-existing `charge_anim` dashed-arc
  also uses screen blend but only renders during active charging, so
  this is the first time the incompatibility surfaces in a steady-state
  user-visible way. The trace itself has full-opacity gradient stroke
  and sits inside a clip-circle on the gradient ring background — a
  plain alpha-blend reads cleanly on both themes without screen.
- Proposed fix: **Drop `mix-blend-mode:screen` from the ECG wrapper
  `<g>`.** Change line 726 from
  `<g clip-path="url(#${ecgClipId})" pointer-events="none" style="mix-blend-mode:screen;">`
  to
  `<g clip-path="url(#${ecgClipId})" pointer-events="none">`.
  Leave the existing `charge_anim` path's `mix-blend-mode:screen` at
  line 752 untouched (separate scope, only renders during active
  charging — out of scope for this fix; can be revisited in a follow-up
  if it ever becomes visible on light themes).
- Verification: manual smoke test on both light and dark HA themes in
  three states: (a) plugged + not charging (flatline must be visible
  on light theme), (b) plugged + actively charging (QRS pulses must
  still read on dark theme), (c) disconnected (ECG group still omitted).

### [nice-to-have] #6 — Comment first-frame regen sentinel idiom
- File: `custom_components/quiet_solar/ui/resources/qs-car-card.js:162-163`
- Severity: nice-to-have
- Source: qs-review-edge-case-hunter + qs-review-blind-hunter
- Description: `(ts - (this._lastEcgRegenTs ?? -Infinity)) / 1000` and
  `Math.abs(this._currentEcgAmp - (this._lastEcgAmp ?? -Infinity))`
  both yield `Infinity` on the very first RAF tick (when the sentinels
  are still `null`), forcing a first-frame regen. A future refactor

  cold-start regen.
- Proposed fix: Add a one-line comment immediately above line 162
  explaining the `-Infinity` sentinels: e.g.
  `// QS-229 - -Infinity sentinels force first-frame regen when _lastEcgRegenTs/_lastEcgAmp are null.`

### [nice-to-have] #7 — Promote bare `320` literal to a module constant
- File: `custom_components/quiet_solar/ui/resources/qs-car-card.js:167, 728`
- Severity: nice-to-have
- Source: qs-review-edge-case-hunter + qs-review-blind-hunter
- Description: `_buildQRSPath(..., ECG_BASELINE_Y, 320)` is called with
  hardcoded `320` (the SVG viewBox width) at both the RAF-regen site
  (line 167) and the initial-render markup (line 728). If the viewBox
  is ever resized, the path will silently fail to span the visible band.
- Proposed fix: Add a module constant alongside the ECG_* block (around
  line 19-32) — e.g. `const ECG_TOTAL_WIDTH_PX = 320;` — and replace
  both call-site literals. Update test_car_card_ecg_tuning_constants_in_band
  to also assert the new constant exists and equals 320.

### [nice-to-have] #8 — Extend ECG guard to skip faulted/stale states

  (or a sibling) asserting the source contains `!isFaulted` and
  `!isStale` inside the ECG conditional.
