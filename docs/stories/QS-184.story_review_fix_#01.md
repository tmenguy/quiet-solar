### [should-fix] S3 — `_parse_frontmatter` fails on CRLF line endings
- File: `tests/qs/agents/test_opencode_agents.py` (`_parse_frontmatter` helper, ~line 80)
- Severity: should-fix
- Source: qs-review-edge-case-hunter
- Description: `text.startswith("---\n")` and `rest.index("\n---\n")` both fail when an agent file is saved with CRLF line endings (e.g., by a Windows contributor or an editor with CRLF defaults). The test then crashes with a misleading assertion message.
- Proposed fix: Normalize line endings at parse time: `text = path.read_text(encoding="utf-8").replace("\r\n", "\n")`. Add a test case covering a CRLF-encoded fixture (use `tmp_path` to write a fixture inline).

### [should-fix] S4 — `_parse_frontmatter` leaks `ValueError` on missing closing delimiter
- File: `tests/qs/agents/test_opencode_agents.py` (`_parse_frontmatter` helper, ~line 82)
- Severity: should-fix
- Source: qs-review-edge-case-hunter
- Description: `rest.index("\n---\n")` raises `ValueError` (not `AssertionError`) when the closing `---` is missing or the file has no trailing newline. The docstring promises `AssertionError` and the failure message becomes unhelpful (`substring not found`).
- Proposed fix: Replace `rest.index(...)` with a guarded find: `end = rest.find("\n---\n")` then `assert end != -1, f"{path}: missing closing --- delimiter"`. Add a test case for a file without a closing delimiter.

### [should-fix] S7 — qs-finish-task remote-branch deletion lacks main/master guard
- File: `.opencode/agents/qs-finish-task.md` (remote-branch deletion blocks, ~lines 105-111 and the duplicate block in Case B)
- Severity: should-fix
- Source: qs-review-coderabbit
- Description: `git push origin --delete "{{branch}}"` has no guard against `main`/`master`. The `.claude/` and `.cursor/` equivalents have similar gaps. A misclassified context (e.g., `branch == "main"`) would silently destroy the default branch.
- Proposed fix: Wrap the deletion in a guard: `if [ "{{branch}}" = "main" ] || [ "{{branch}}" = "master" ]; then echo "refusing to delete protected branch"; exit 1; fi` (or the YAML equivalent for the agent body). Apply to BOTH occurrences (Case A "no PR" and Case B "PR merged"). Mirror the same guard in `.claude/agents/qs-finish-task.md` and `.cursor/agents/qs-finish-task.md` for parity.
