### [must-fix] `http.client.InvalidURL` crashes the script instead of falling back
- File: `scripts/qs/spawn_session.py:169`
- Severity: must-fix
- Source: qs-review-edge-case-hunter
- Description: `_api` catches `(HTTPError, URLError, OSError)` but `http.client.InvalidURL` does not subclass any of these. A malformed `OPENCODE_SERVER_PORT` (e.g. `"4096 "` with trailing whitespace, or `"abc"`), or any control character in `session_id`, causes `urllib.request` to raise `InvalidURL` and the script crashes with an uncaught stack trace instead of routing through the CLI fallback path.
- Proposed fix: Extend the except tuple in `_api` to include `http.client.InvalidURL` (or use the broader `http.client.HTTPException` superclass). Add a regression test `test_invalid_url_falls_back` that monkeypatches `urllib.request.urlopen` to raise `http.client.InvalidURL` and asserts the fallback-CLI path is taken (exit 0 when `opencode` on PATH, exit 2 otherwise).

### [must-fix] Shell-injection vector via `issue` parameter in `_opencode_cli_command`
- File: `scripts/qs/launchers/opencode.py` (`_opencode_cli_command`, around the `Path(tempfile.gettempdir()) / f"qs_oc_launch_{issue}.sh"` line and the `f"sh {script_path}"` return)
- Severity: must-fix
- Source: qs-review-edge-case-hunter
- Description: The signature `issue: int | str` admits string values that contain shell metacharacters. Both the on-disk filename and the returned `new_context` interpolate `issue` raw. A caller passing `issue="42; rm -rf $HOME"` produces a script file at `/tmp/qs_oc_launch_42; rm -rf $HOME.sh` and a `new_context` of `sh /tmp/qs_oc_launch_42; rm -rf $HOME.sh` — the `;` terminates the `sh` invocation and the second clause executes. The in-pipeline caller passes an int, but the type contract is the contract.
- Proposed fix: At the top of `_opencode_cli_command`, coerce `issue_int = int(issue)` (raising `ValueError` on failure) and use `issue_int` for the filename — narrows the public contract to integers. Additionally, wrap the returned path in `shlex.quote`: `return f"sh {shlex.quote(str(script_path))}"`. Add tests `test_opencode_cli_command_rejects_non_integer_issue` and `test_opencode_cli_command_shell_quotes_script_path` (parametrize with a `TMPDIR` that contains whitespace).

### [should-fix] `_opencode_cli_command` writes a deterministic shared `/tmp` path
- File: `scripts/qs/launchers/opencode.py` (`_opencode_cli_command`)
- Severity: should-fix
- Source: qs-review-blind-hunter + qs-review-edge-case-hunter
- Description: `Path(tempfile.gettempdir()) / f"qs_oc_launch_{issue}.sh"` is a deterministic path keyed only on issue. Concurrent setup-task invocations race on the file; tests calling `build_payload` repeatedly mutate the same artifact; on a multi-user host a malicious user could symlink the path to a sensitive file (e.g. `~/.ssh/authorized_keys`) before the script is written.
- Proposed fix: Use `tempfile.NamedTemporaryFile(mode="w", prefix=f"qs_oc_launch_{issue}_", suffix=".sh", delete=False)` and `chmod(0o700)` on the resulting path. Capture `f.name` for `script_path`. Test: `test_opencode_cli_command_uses_unique_paths` calls `_opencode_cli_command` twice with the same issue and asserts the two script paths differ.

### [should-fix] Agent name interpolated raw into single-quoted shell echo
- File: `scripts/qs/launchers/opencode.py` (`_opencode_cli_command`, the `echo '── Activating agent: {agent} ──'` line in the generated script)
- Severity: should-fix
- Source: qs-review-blind-hunter
- Description: The echo line uses `f"echo '── Activating agent: {agent} ──'"` — interpolating `agent` raw inside a single-quoted shell literal. If `agent` ever contains a single quote or backslash, the generated script breaks. Inconsistent with the surrounding `shlex.quote` calls.
- Proposed fix: Replace with `echo {shlex.quote(f'── Activating agent: {agent} ──')}`. Add a parametrized test case that passes an `agent` value containing a single quote and asserts the generated script is syntactically valid via `subprocess.run(["bash", "-n", script_path])`.

### [should-fix] Tempfile write/chmod failure propagates uncaught + whitespace `TMPDIR` breaks return
- File: `scripts/qs/launchers/opencode.py` (`_opencode_cli_command`)
- Severity: should-fix
- Source: qs-review-edge-case-hunter + qs-review-blind-hunter
- Description: `Path.write_text` and `chmod(0o755)` raise `OSError` / `PermissionError` on read-only `/tmp`, non-POSIX filesystems (Windows shares, sandboxed CI), and the failure escapes `_opencode_cli_command` → `build_payload` → `setup_task.main()` as an uncaught stack trace with no JSON error payload. Separately, when `TMPDIR=/tmp/with space/` the unquoted `f"sh {script_path}"` returns `sh /tmp/with space/qs_oc_launch_42.sh`, which the shell splits on whitespace.
- Proposed fix: Wrap the write + chmod in `try/except OSError`; on failure, fall back to inlining the full `opencode <worktree> --agent <name> --prompt <kickoff>` command directly into `new_context` instead of writing a script file (skipping the wrapper script entirely). And apply `shlex.quote(str(script_path))` to the returned `sh` invocation (this is also covered by the must-fix #2 fix). Add tests: (a) monkeypatch `Path.write_text` to raise `OSError` and assert the inline-command fallback is returned; (b) monkeypatch `tempfile.gettempdir` to return `/tmp/with space` and assert the returned string is shell-safe.

### [should-fix] `session_id` not URL-escaped before path interpolation
- File: `scripts/qs/spawn_session.py` (around lines 237 and 246, `f"/session/{session_id}/prompt_async"` and `f"/session/{session_id}"`)
- Severity: should-fix
- Source: qs-review-edge-case-hunter
- Description: `session_id` from the server response is interpolated directly into URL paths without escaping. A server returning `{"id": "abc/../delete-me"}` causes path traversal against the OpenCode API; `?` truncates the path; control characters trigger the unhandled `InvalidURL` covered by must-fix #1. Even with the `InvalidURL` fix, escaping is the correct defense.
- Proposed fix: `from urllib.parse import quote` at the top of `spawn_session.py`. Pre-encode: `safe_id = quote(str(session_id), safe="")` and use `safe_id` in the path interpolations. Add `test_spawn_session_url_escapes_session_id` that mocks the server to return `{"id": "abc/../delete-me"}` and asserts the second `urlopen` call's URL contains `abc%2F..%2Fdelete-me` not `abc/../delete-me`.

### [should-fix] Brittle module/path teardown in `tests/qs/conftest.py`
- File: `tests/qs/conftest.py:49-52`
- Severity: should-fix
- Source: qs-review-coderabbit (corroborated by qs-review-blind-hunter)
- Description: The autouse fixture uses substring match `path_str in mod_file` to decide which modules to purge — this can remove unrelated modules whose `__file__` happens to contain the path string. Separately, `sys.path.remove(path_str)` raises `ValueError` if another fixture/test already removed the entry.
- Proposed fix: Compute `scripts_qs_dir = Path(path_str).resolve()`. For each entry in `sys.modules`, resolve `Path(mod.__file__).resolve()` (wrapped in `try/except (OSError, AttributeError, RuntimeError)`) and use `mod_path.is_relative_to(scripts_qs_dir)` (Py3.9+). Guard `sys.path.remove(path_str)` with `if path_str in sys.path:` or wrap in `try/except ValueError`.
