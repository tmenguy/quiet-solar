### [should-fix] Hardcoded user-originated marker keys
- File: `custom_components/quiet_solar/ha_model/car.py:732-735`
- Severity: should-fix
- Source: qs-review-coderabbit
- Description: The new `is_manual` expression in `_update_car_api_staleness`
  hardcodes the marker keys `"charger_name"` and `"car_name"`. Repo
  guideline: config/marker keys belong in `const.py`, never as bare
  strings. The same literals are the pre-existing convention across
  `charger.py`/`car.py` (~25+ usages), so the proper fix introduces the
  constants and migrates the literals.
- Proposed fix: Add to `custom_components/quiet_solar/const.py`:
  `USER_ORIGINATED_CHARGER_NAME: str = "charger_name"` and
  `USER_ORIGINATED_CAR_NAME: str = "car_name"`. Replace every
  `get_user_originated` / `set_user_originated` / `clear_user_originated`
  call that passes the bare `"charger_name"` / `"car_name"` literals in
  `custom_components/quiet_solar/ha_model/car.py` and
  `custom_components/quiet_solar/ha_model/charger.py` (and any other
  production `.py`) with the new constants. Update test files that pass
  these marker keys (e.g. `tests/test_car_api_staleness.py`
  `set_user_originated("charger_name", ...)`) to import and use the
  constants. Pure mechanical substitution â€” no behavior change; the
  constants' values must remain exactly `"charger_name"` / `"car_name"`
  because they are persisted in stored user-originated state.

### [nice-to-have] Single source of truth for the manual-origin definition
- File: `custom_components/quiet_solar/ha_model/car.py:730-735`
- Severity: nice-to-have
- Source: qs-review-blind-hunter
- Description: The "what counts as manual" definition lives twice: the
  `is_manual` OR-expression at the periodic call site and the implicit
  `manual=True` literal at the user entry point. The docstring argues
  they are equivalent by construction, but a helper removes the drift
  risk outright.
- Proposed fix: Extract a private helper on the car, e.g.
  `def _charger_assignment_is_user_originated(self) -> bool:` returning
  the existing OR-expression (requires `self.charger is not None`;
  keep the existing guard at the call site or assert/early-return
  `False` when `self.charger is None`). Call it from
  `_update_car_api_staleness`; move the "equivalent by construction"
  docstring rationale onto the helper. The entry point keeps the
  literal `manual=True` (it is definitionally manual). Add/adjust no
  tests beyond what coverage requires â€” the three periodic
  origin-detection tests already cover both branches of the helper.
