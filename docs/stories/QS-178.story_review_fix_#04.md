### [must-fix] `_battery_get_charging_power` runs on every positive-delta `_constraints_delta` call
- File: `custom_components/quiet_solar/home_model/solver.py:608` (entry-seed) + `:484, 521` (reseed)
- Severity: must-fix (CRITICAL regression from round-3 must-fix #1)
- Source: qs-review-blind-hunter

#### Description

Round-3 must-fix #1 threaded `battery_commands` into `_constraints_delta`'s
entry-seed and per-placement reseed. The implementation used an
overly broad entry guard:

```python
if energy_delta > 0 and self._battery is not None:
    bat_charge_traj = self._battery_get_charging_power(
        existing_battery_commands=battery_commands
    )[1].copy()
```

This fires on EVERY positive-delta call to `_constraints_delta` —
including mandatory allocation, legacy paths, and all 49+ existing
call sites that did NOT exist for the floor-guard feature. The
resulting `bat_charge_traj` is passed to `adapt_repartition`, where
`use_battery_floor = True` (because `bat_charge_traj is not None`).
With `battery_min_wh=0.0` (default for legacy callers), the clamp:

```python
max_drain_wh = max(0.0, forward_min[i] - 0.0) = forward_min[i]
```

still fires whenever the simulated trajectory shows the battery at a
low charge (e.g., `forward_min[i]` ≈ 100 Wh at some slot). Legacy
callers that previously made placements unconditionally now get
clamped — silently changing solver behavior across the entire system.

#### Proposed fix

Gate the entry-seed on `battery_min_wh > 0` so legacy callers (which
pass `battery_min_wh=0.0` by default) keep their pre-QS-178 behavior:

```python
# at the entry of _constraints_delta, where bat_charge_traj is seeded:
if battery_min_wh > 0 and energy_delta > 0 and self._battery is not None:
    bat_charge_traj = self._battery_get_charging_power(
        existing_battery_commands=battery_commands
    )[1].copy()
else:
    bat_charge_traj = None
```

Apply the same gate at the per-placement reseed (lines ~484 / ~521):

```python
# at the per-placement reseed:
if battery_min_wh > 0 and self._battery is not None and has_changes:
    bat_charge_traj = self._battery_get_charging_power(
        existing_battery_commands=battery_commands
    )[1].copy()
```

The surplus block calls `_constraints_delta` with
`battery_min_wh = self._battery.get_value_empty() + safety_margin_wh
> 0`, so it gets full Layer-3 semantics. All other callers default
`battery_min_wh=0.0` and get pre-QS-178 semantics.

#### Tests to add

`tests/test_solver_pre_discharge.py`:
- `test_constraints_delta_does_not_inject_trajectory_for_legacy_callers`:
  - Build a scenario WITHOUT the surplus block firing (e.g., low
    waste, battery near empty).
  - Patch `_battery_get_charging_power` to record its calls; tag each
    with the caller stack frame (e.g., via `inspect.stack()`).
  - Run `solve()`.
  - Assert that NO call to `_battery_get_charging_power` originated
    from `_constraints_delta` when `battery_min_wh = 0` (or
    equivalently: the only call coming from `_constraints_delta` is
    when called from the surplus block).

`tests/test_coverage_constraints.py` (or similar broad suite): run
existing tests as-is; they should pass without modification (the
fix restores their previously-relied-on behavior).

### [must-fix] Multi-pass skip still gates off legacy upgrade path
- File: `custom_components/quiet_solar/home_model/constraints.py:1471`
- Severity: must-fix (tightly coupled with must-fix #1)
- Source: qs-review-blind-hunter

#### Description

The multi-pass skip `if use_battery_floor and out_commands[i] is not None: continue`
correctly gates the double-decrement hazard when `use_battery_floor=True`.
But with must-fix #1 present (entry-seed firing on every positive-delta
call), `use_battery_floor` is True for all 49+ legacy callers, gating
off pass-2's legacy upgrade path for them too.

Fixing must-fix #1 (gate entry-seed on `battery_min_wh > 0`) also
indirectly fixes this: legacy callers no longer get `bat_charge_traj`
injected, so `use_battery_floor=False`, and the multi-pass skip
becomes a no-op for them.

**Defense-in-depth**: also strengthen the multi-pass skip's gate to
require both `use_battery_floor` AND `battery_min_wh > 0` (or
equivalently, just `battery_min_wh > 0`), so even if a future change
re-introduces the over-broad entry-seed, the multi-pass skip is
contained:

```python
if use_battery_floor and battery_min_wh > 0 and out_commands[i] is not None:
    continue
```

### [must-fix] Story Task 8 pseudocode + Dev Notes still show closed-window decrement
- File: `docs/stories/QS-178.story.md:521-522, 766-769`
- Severity: must-fix
- Source: qs-review-acceptance-auditor

#### Description

Round-3 fix #02 updated AC-6 (story lines 153-154) to the open-ended
`[i:]` decrement. The implementation matches. But Task 8's pseudocode
at story.md:521-522 and Dev Notes at story.md:766-769 still describe
the old closed-window form `[i:last_slot+1]`.

This contradicts the implementation and undermines the "story is
source of truth" policy.

#### Proposed fix

In `docs/stories/QS-178.story.md`:

**Task 8 pseudocode (lines 521-522)** — update from:
```text
bat_charge_traj[i:last_slot + 1] -= state_delta_wh
forward_min[i:last_slot + 1]     -= state_delta_wh
```
to:
```text
bat_charge_traj[i:]              -= state_delta_wh
forward_min[i:]                  -= state_delta_wh
# Then back-propagate forward_min into [first_slot, i) via
# _back_propagate_forward_min to keep the multi-pass invariant.
```

**Dev Notes (lines 766-769)** — replace the closed-window invariant
description with the open-ended form + a sentence explaining why:

```markdown
The decrement is **open-ended** (`[i:]`, not `[i:last_slot+1]`)
because physical battery charge carries forward across the
constraint's `last_slot` boundary into all subsequent slots.
Downstream constraints in the same `_constraints_delta` call (before
the cross-call reseed) must see the correct post-placement physical
trajectory. The multi-pass `forward_min` invariant is preserved by
the `_back_propagate_forward_min` helper which propagates the new
minimum back into `[first_slot, i)`.
```

### [should-fix] Clamp branch lacks defensive assert after `delta_power` recompute
- File: `custom_components/quiet_solar/home_model/constraints.py:1685`
- Severity: should-fix
- Source: qs-review-blind-hunter

#### Description / Fix

The clamp's recompute path produces `delta_power = base_cmd.power_consign
- current_command_power (+ piloted_offset)`. The arithmetic invariant
`delta_power <= max_delta_power_floor` holds today (since
`allowed_base_power = current + floor - piloted_offset`, then

```python

    f"clamp violated: delta_power={delta_power:.3f} > "
    f"max_delta_power_floor={max_delta_power_floor:.3f}"
)
```

(1e-9 tolerance handles float arithmetic noise.)

### [should-fix] Weak smoke test for `battery_commands` threading
- File: `tests/test_solver_pre_discharge.py:1374-1460`
- Severity: should-fix
- Source: qs-review-blind-hunter + qs-review-edge-case-hunter + qs-review-acceptance-auditor

#### Description / Fix

`test_constraints_delta_uses_battery_commands_for_floor_guard`'s
assertions are both `any(sim_calls)` — trivially true because the
pre-surplus refresh at solver.py:572 ALWAYS calls
`_battery_get_charging_power(existing_battery_commands=battery_commands)`.
The test passes even if the new threading at lines 484/521 silently
regresses.

#### Proposed fix

Flag the surplus-block-active context explicitly and assert per-call:

```python
import contextlib

# Track when we're inside _constraints_delta originating from the
# surplus block path.
surplus_block_active = [False]
sim_calls_in_surplus_block = []  # list of `existing_battery_commands` per call

_real_get_power = s._battery_get_charging_power

def _spy_get_power(*args, **kwargs):
    if surplus_block_active[0]:
        sim_calls_in_surplus_block.append(kwargs.get("existing_battery_commands"))
    return _real_get_power(*args, **kwargs)

_real_constraints_delta = s._constraints_delta

def _wrap_constraints_delta(*args, **kwargs):
    # Detect surplus-block invocation: battery_min_wh > 0 is the
    # distinguishing kwarg per must-fix #1's gate.
    is_surplus = kwargs.get("battery_min_wh", 0.0) > 0
    if is_surplus:
        surplus_block_active[0] = True
    try:
        return _real_constraints_delta(*args, **kwargs)
    finally:
        if is_surplus:
            surplus_block_active[0] = False

with patch.object(s, "_battery_get_charging_power", side_effect=_spy_get_power), \
     patch.object(s, "_constraints_delta", side_effect=_wrap_constraints_delta):
    s.solve()

assert len(sim_calls_in_surplus_block) >= 1, (
    "Surplus block must invoke _battery_get_charging_power internally "
    "(both entry-seed and per-placement reseed)."
)
for call_kwargs in sim_calls_in_surplus_block:
    assert call_kwargs is not None, (
        "_constraints_delta inside the surplus block must pass "
        "existing_battery_commands=battery_commands."
    )
```

### [should-fix] Drop `probe_window_end > probe_window_start` gate on post-shrink recompute
- File: `custom_components/quiet_solar/home_model/solver.py:1482-1490`
- Severity: should-fix
- Source: qs-review-edge-case-hunter

#### Description / Fix

if probe_window_end >= probe_window_start:  # broad guard; degenerate-window check follows

(Then the degenerate-window guard at the post-shrink site catches

### [should-fix] Slot-count expansion missing negative clamp
- File: `tests/test_solver.py:884-888`
- Severity: should-fix
- Source: qs-review-edge-case-hunter

#### Description / Fix

negative values if `cmd_start > cmd_end` (e.g., unordered commands, or
