# QS-173 Review Fix #05

Phase 4 re-review of PR #174 (after fix #04) surfaced 1 must-fix and 6
should-fix items. MF-1 is a regression introduced by fix #04 itself
(the SF-4 fix's `2>&1` capture now mixes SSH banners into the SHA
pipeline). User selected option (d): land all must-fix + all
should-fix. Nice-to-haves are explicitly out of scope.

## Findings to address

### must-fix

**MF-1 (`ls-remote 2>&1` mixes stderr into REMOTE_TIP):** at
`scripts/worktree-setup.sh:285`, fix #04's SF-4 captured `2>&1` to
surface failures. But any SSH banner / MOTD / known_hosts warning /
X11-forwarding warning on a *successful* `ls-remote` interleaves with
the SHA line, and `awk '{print $1}'` then emits the first column of
every line → `REMOTE_TIP` becomes multi-line garbage. Downstream
`cat-file -e`, fetch retry, and `merge-base --is-ancestor` all
misbehave; the operator gets a bogus "non-fast-forward" error. This
hits common GitHub SSH setups with any stale known_hosts entry.

### should-fix

**SF-1 (fetch retry still `2>/dev/null`):** at
`scripts/worktree-setup.sh:303`, the fetch retry inside the
divergence-check ancestry resolution swallows stderr the way the
original `ls-remote` did before SF-4. Same class of bug — operator sees
"failed to fetch" but never the actual `fatal: …` line.

**SF-2 (`merge-base --is-ancestor 2>/dev/null` collapses rc=1 vs rc>1):**
at `scripts/worktree-setup.sh:316`, exit code 1 ("not ancestor") and
>1 (bad ref, broken repo) collapse into the same "non-fast-forward"
message. Apply the same exit-code distinction pattern that SF-4 added
for ls-remote.

**SF-3 (`find` exclusion misses Python project artifacts):** at
`scripts/worktree-setup.sh:143-155`, the exclusion list covers IDE/OS
dotfiles but not `.venv`, `__pycache__`, `.pytest_cache`,
`.mypy_cache`, `.ruff_cache`, `.tox`, or `node_modules`. A stale
untracked directory from a prior Python session is full of these and
blocks recovery as "non-git content".

**SF-4 (`ACTUAL_HEAD || echo ""` swallows rev-parse failure):** at
`scripts/worktree-setup.sh:76`, if `.git` is corrupt or its file
pointer points at a missing main-repo worktree entry, rev-parse
silently returns empty. The `git status` backstop at L121 catches most
cases but not all (e.g. `.git` is a *file* with a bad gitdir pointer).

**SF-5 (cleanliness gate skips HAS_NONGIT_CONTENT when
IS_TRACKED_WORKTREE=yes but `.git` is missing):** at
`scripts/worktree-setup.sh:156-157`, if porcelain lists the worktree
as tracked but `.git` was manually removed, DIRTY/STASHES are skipped
(L117 false) and the HAS_NONGIT_CONTENT clause is gated on
`IS_TRACKED_WORKTREE="no"`. Gate passes; `worktree remove --force`
removes user content silently.

**SF-6 (post-recovery rebuild may collide with diverged remote):**
partial-state recovery removes the worktree, then `git worktree add`
rebuilds. If `refs/heads/QS_<N>` was also deleted, the rebuild creates
the branch at `origin/main` tip. If `origin/QS_<N>` has commits not in
`origin/main`, the new divergence check refuses to push → recovery
loops permanently.

## Out of scope (explicitly NOT in this fix)

- All nice-to-haves: coderabbit MD040 on fix-plan #04 line 293, stash
  list repo-wide (acknowledged OOS multiple times), redundant
  `[ -d "$WORKTREE_DIR" ]` at L143, fork-remote upstream override,
  `find | head` SIGPIPE under `set -euo pipefail`, detached-HEAD
  in-progress-rebase wording, truncation indicator on `HAS_NONGIT_CONTENT`,
  settings.json allowlist deduplication.

## Plan

### Task A — Separate stdout/stderr for `ls-remote` (`scripts/worktree-setup.sh`)

**A.1 (MF-1)** — Replace the `2>&1` capture pattern (currently L285)
with a two-channel capture. Use a temp file for stderr; treat stdout
as authoritative for the SHA pipeline; print stderr only on failure.

```bash
LS_REMOTE_STDERR="$(mktemp -t worktree-setup.XXXXXX)"
trap 'rm -f "$LS_REMOTE_STDERR"' EXIT
if ! REMOTE_TIP_OUTPUT="$(git -C "$WORKTREE_DIR" ls-remote --heads origin "$BRANCH" 2>"$LS_REMOTE_STDERR")"; then
    echo "Error: 'git ls-remote --heads origin ${BRANCH}' failed."
    echo "  Stderr: $(cat "$LS_REMOTE_STDERR")"
    echo "Cannot verify divergence before push. Resolve manually:"
    echo "  cd ${WORKTREE_DIR} && git ls-remote --heads origin ${BRANCH}"
    exit 1
fi
REMOTE_TIP="$(echo "$REMOTE_TIP_OUTPUT" | awk '{print $1}')"
# Empty REMOTE_TIP now genuinely means "branch doesn't exist on origin",
# not "ls-remote failed".
```

Note: the trap is intentionally script-wide. If other code paths add
more temp files later, refactor to a single cleanup function. The
`-t worktree-setup.XXXXXX` template gives the file a stable prefix for
debugging.

### Task B — Stop swallowing fetch stderr (`scripts/worktree-setup.sh`)

**B.1 (SF-1)** — At the fetch retry inside the ancestry check
(currently L303), capture stderr the same way and surface it on
failure:

```bash
FETCH_STDERR="$(mktemp -t worktree-setup.XXXXXX)"
# Add to the existing EXIT trap: trap 'rm -f "$LS_REMOTE_STDERR" "$FETCH_STDERR"' EXIT
if ! git -C "$WORKTREE_DIR" cat-file -e "${REMOTE_TIP}^{commit}" 2>/dev/null; then
    echo "Fetching origin/${BRANCH} to compare ancestry..."
    if ! git -C "$WORKTREE_DIR" fetch origin "$BRANCH" 2>"$FETCH_STDERR"; then
        echo "Error: failed to fetch origin/${BRANCH} for divergence check."
        echo "  Stderr: $(cat "$FETCH_STDERR")"
        echo "Network or remote-access issue. Resolve manually:"
        echo "  cd ${WORKTREE_DIR} && git fetch origin ${BRANCH}"
        exit 1
    fi
fi
```

The `cat-file -e` test stays with `2>/dev/null` because it's a probe —
non-zero just means "not in local objects".

### Task C — Distinguish merge-base error codes (`scripts/worktree-setup.sh`)

**C.1 (SF-2)** — Replace the boolean `if ! merge-base --is-ancestor`
with an explicit exit-code check:

```bash
MB_STDERR="$(mktemp -t worktree-setup.XXXXXX)"
# Add to EXIT trap.
git -C "$WORKTREE_DIR" merge-base --is-ancestor "$REMOTE_TIP" "$LOCAL_TIP" 2>"$MB_STDERR"
MB_RC=$?
if [ $MB_RC -eq 0 ]; then
    : # remote tip is ancestor of local → fast-forward push is safe
elif [ $MB_RC -eq 1 ]; then
    # genuine non-fast-forward
    echo "Error: origin/${BRANCH} has commits not in local ${BRANCH}."
    echo "  Remote tip: ${REMOTE_TIP}"
    echo "  Local tip:  ${LOCAL_TIP}"
    echo "Refusing to push (would be non-fast-forward)."
    echo "Resolve manually:"
    echo "  cd ${WORKTREE_DIR} && git fetch && git rebase origin/${BRANCH}"
    echo "…then re-run this script (or just 'git push')."
    exit 1
else
    # rc > 1 → genuine git error (bad ref, broken repo)
    echo "Error: 'git merge-base --is-ancestor' failed (rc=${MB_RC})."
    echo "  Stderr: $(cat "$MB_STDERR")"
    echo "  Remote tip: ${REMOTE_TIP}"
    echo "  Local tip:  ${LOCAL_TIP}"
    echo "Cannot verify divergence. Resolve manually:"
    echo "  cd ${WORKTREE_DIR} && git log --oneline ${LOCAL_TIP} ${REMOTE_TIP}"
    exit 1
fi
```

Note: `set -e` won't fire on the bare `merge-base` call because we
immediately capture `$?` before the next command. To be extra safe,
prefix with `set +e` … `set -e`:

```bash
set +e
git -C "$WORKTREE_DIR" merge-base --is-ancestor "$REMOTE_TIP" "$LOCAL_TIP" 2>"$MB_STDERR"
MB_RC=$?
set -e
```

### Task E — Surface rev-parse failure (`scripts/worktree-setup.sh`)

**E.1 (SF-4)** — At ACTUAL_HEAD capture (L76), replace the silent
fallback with explicit failure handling:

```bash
RP_STDERR="$(mktemp -t worktree-setup.XXXXXX)"
# Add to EXIT trap.
if ! ACTUAL_HEAD="$(git -C "$WORKTREE_DIR" rev-parse --abbrev-ref HEAD 2>"$RP_STDERR")"; then
    echo "Error: 'git rev-parse --abbrev-ref HEAD' failed in ${WORKTREE_DIR}."
    echo "  Stderr: $(cat "$RP_STDERR")"
    echo "The worktree's git metadata may be corrupted or the main repo's"
    echo "worktree entry for it may have been removed. Resolve manually:"
    echo "  cd ${WORKTREE_DIR} && cat .git   # inspect gitdir pointer"
    echo "  ls -la ${MAIN_DIR}/.git/worktrees/"
    exit 1
fi
```

Apply the same pattern to `ACTUAL_UPSTREAM` (currently L92): rev-parse
of `@{u}` failing means the branch genuinely has no upstream, which is
expected for a freshly-created local branch — so for `@{u}` keep the
`|| echo ""` form (a missing upstream is legitimate state). The
critical change is for HEAD rev-parse.

### Task F — Gate HAS_NONGIT_CONTENT in tracked-no-.git path (`scripts/worktree-setup.sh`)

**F.1 (SF-5)** — Modify the cleanliness-gate condition (currently
L156-157) to also consult HAS_NONGIT_CONTENT when `.git` is missing
even on a tracked worktree:

```bash
# Gate destructive recovery if:
#   - the worktree has uncommitted edits (DIRTY),
#   - any stash exists in this repo (STASHES, repo-wide),
#   - or (no .git, regardless of tracked-status) the directory has
#     non-git user content the operator wouldn't expect us to wipe.
NO_GIT_PRESENT="no"
if [ ! -d "${WORKTREE_DIR}/.git" ] && [ ! -f "${WORKTREE_DIR}/.git" ]; then
    NO_GIT_PRESENT="yes"
fi
if [ -n "$DIRTY" ] || [ -n "$STASHES" ] || \
   { [ "$NO_GIT_PRESENT" = "yes" ] && [ -n "$HAS_NONGIT_CONTENT" ]; }; then
    echo "Error: ${WORKTREE_DIR} has uncommitted changes, stashes, or non-git content."
    echo "  Status:  ${DIRTY:-(none)}"
    echo "  Stashes: ${STASHES:-(none)}"
    echo "  Other:   ${HAS_NONGIT_CONTENT:-(none)}"
    echo "Refusing to destructively recover. Resolve manually:"
    echo "  ls -la ${WORKTREE_DIR}"
    exit 1
fi
```

Effectively: tracked + present-.git → trust DIRTY/STASHES (status
already saw everything); tracked + missing-.git → fall back to
HAS_NONGIT_CONTENT (since DIRTY/STASHES are unknowable); untracked →
HAS_NONGIT_CONTENT applies the same way.

### Task G — Break the post-recovery divergence loop (`scripts/worktree-setup.sh`)

**G.1 (SF-6)** — Track whether this run rebuilt the worktree, and
relax the divergence check in that case:

```bash
# Near the top of the directory-present block:
WORKTREE_RECOVERED="no"

# In the recovery arms (both `worktree remove --force` and `rm -rf` paths):
WORKTREE_RECOVERED="yes"

# At the divergence check (currently L285+):
if [ "$WORKTREE_RECOVERED" = "yes" ]; then
    # If we just rebuilt the worktree (worktree remove --force or rm -rf),
    # the local branch may have been recreated from origin/main and would
    # falsely appear to be "behind" origin/${BRANCH}. Reset local to match
    # origin/${BRANCH} before pushing, so push -u becomes a no-op upload.
    if [ -n "$REMOTE_TIP" ]; then
        echo "Worktree was rebuilt; resetting local ${BRANCH} to match origin/${BRANCH}..."
        if ! git -C "$WORKTREE_DIR" fetch origin "$BRANCH" 2>/dev/null; then
            echo "Warning: fetch of origin/${BRANCH} failed; divergence check may misbehave."
        fi
        # Reset local branch to remote tip so push is a no-op rather than non-FF.
        if ! git -C "$WORKTREE_DIR" reset --hard "origin/$BRANCH"; then
            echo "Error: failed to reset local ${BRANCH} to origin/${BRANCH}."
            echo "Resolve manually:"
            echo "  cd ${WORKTREE_DIR} && git reset --hard origin/${BRANCH}"
            exit 1
        fi
    fi
fi
# …then existing divergence check…
```

Rationale: when this run rebuilt the worktree, we deliberately threw
away local state — there's no operator work to preserve, so resetting
to the remote tip is the right move (no commits get lost). The
divergence check still runs (catches the no-rebuild case where local
branch had real divergence).

Acceptable alternative for the implementer: skip the divergence check
entirely when `WORKTREE_RECOVERED="yes"` (since the rebuild path
guarantees local was just created from a known-good remote ref). Pick
whichever is simpler in the resulting code.

## Acceptance criteria

- **AC-1:** `ls-remote` stdout and stderr are captured to separate
  channels; the SHA-extraction pipeline reads only stdout; stderr is
  reported only on `ls-remote` failure.
- **AC-2:** the fetch retry in the divergence-check ancestry
  resolution surfaces its actual stderr output on failure.
- **AC-3:** `merge-base --is-ancestor` distinguishes rc=0
  (fast-forward), rc=1 (non-fast-forward), and rc>1 (real git error)
  with different diagnostic messages and remediation hints.
- **AC-4:** the `find` exclusion list covers `.venv`, `__pycache__`,
  `.pytest_cache`, `.mypy_cache`, `.ruff_cache`, `.tox`, and
  `node_modules` in addition to the existing IDE/OS dotfiles.
- **AC-5:** `git rev-parse --abbrev-ref HEAD` failure for ACTUAL_HEAD
  is surfaced with its stderr; the script exits with a clear
  "metadata corrupted / main worktree pointer missing" hint rather
  than silently falling through.
- **AC-6:** the cleanliness gate refuses destructive recovery on a
  tracked-by-porcelain worktree whose `.git` is missing if non-git
  user content is present in the directory.
- **AC-7:** when this run rebuilt the worktree (via `worktree remove
  --force` or `rm -rf`), the divergence check does not refuse-and-loop
  against a pre-existing diverged `origin/$BRANCH`; either skip the
  divergence check or reset local to match the remote before pushing.
- **AC-8:** all ACs from the parent story, fix #01, fix #02, fix #03,
  and fix #04 still hold (no regressions).

## Implementer notes

- `/implement-setup-task` is correct: only `scripts/worktree-setup.sh`
  is touched.
- Consolidate temp-file handling: set up a single `TMPFILES=()` array
  + `trap 'rm -f "${TMPFILES[@]}"' EXIT` at the top of the script, and
  push each `mktemp` result into the array. Avoids the "each task
  overwrites the previous trap" foot-gun.
- For Task C, the `set +e ... set -e` pattern around the bare
  `merge-base` invocation is the safest form. The alternative `git
  ... || MB_RC=$?` is shorter but `pipefail` interactions with `set -e`
  inside command substitutions can be subtle.
- For Task G, the `reset --hard origin/$BRANCH` approach is preferred
  over "skip divergence check entirely" because it preserves the
  invariant that after this script, local matches origin (so subsequent
  ops like `git pull` are no-ops). The "skip divergence" alternative
  is acceptable but leaves a less consistent state.
- For Task E, do **not** apply the strict-failure pattern to `@{u}`
  rev-parse — a fresh local branch with no upstream is a legitimate
  state (the script will set it up on the push). Strict-failure there
  would refuse to bootstrap fresh worktrees.
- Total expected diff: ~80-100 lines added, with a net consolidation
  of stderr-handling patterns across Tasks A/B/C/E.
