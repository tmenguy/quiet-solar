## Findings to fix

### [should-fix] S1 — concurrent temp-name race in atomic write
- File: `custom_components/quiet_solar/ui/dashboard.py:424` (`_async_atomic_write_bytes`) + `:527` (`async_update_resources` except)
- Severity: should-fix
- Source: qs-review-edge-case-hunter
- Scope: NEW — introduced by round-1 S12 atomic-write fix
- Description: `async_update_resources` is awaited from two places — startup
  (`async_register_dashboards`) and the "Generate Dashboard" button
  (`generate_dashboard_yaml`) — with no lock. The atomic write uses a
  shared, non-unique temp name `{dst}.qstmp`. If the two invocations
  interleave: (a) interleaved writes corrupt the temp before either
  `os.replace`; (b) writer A's `os.replace(tmp, dst)` raises
  `FileNotFoundError` because writer B already renamed the shared `.qstmp`
  away. That `FileNotFoundError` is caught by the `except FileNotFoundError`
  at line 527 (intended for a missing SOURCE directory), so the remaining
  files are silently skipped and mis-logged as "No bundled resources
  directory found."
- Proposed fix: Make the temp name unique per write (e.g. include
  `os.getpid()` + a counter, or the `tag`, or use `tempfile.mkstemp` in the
  destination dir). AND narrow the `except FileNotFoundError` at line 527 so
  a per-file replace race can't be mistaken for a missing source tree
  (scope it to the source-listing call only). Add a test simulating the
  interleave (or at least asserting the temp name is unique per call).

### [should-fix] S2 — car card's local `parseTimeToMinutes` not hardened
- File: `custom_components/quiet_solar/ui/resources/qs-car-card.js:982`
- Severity: should-fix
- Source: qs-review-edge-case-hunter
- Scope: migration consistency gap
- Description: The car was migrated to extend `QsCardBase`, whose hardened
  `_parseTimeToMinutes` (round-1 S16) returns the 07:00 fallback for invalid
  states. But the car still has its own local closure `parseTimeToMinutes`
  that only guards `if (!txt)` — so `'unavailable'`/`'unknown'` (truthy) →
  `Number('unavailable')` → NaN → `0` → midnight, driving the ring/handle to
  00:00. The migration's whole point was to share the hardened helper.
- Proposed fix: Replace the car's local closure with the inherited
  `this._parseTimeToMinutes`, or add the `_isValidState` guard so invalid
  states fall back to the documented default. Reproduces when the car's
  `next_charge` time entity is `unavailable`/`unknown` at render. Extend the
  per-card test to cover the car's time parsing for invalid states.

### [should-fix] S3 — escape HA-controlled strings before innerHTML (car)
- File: `custom_components/quiet_solar/ui/resources/qs-car-card.js:1166-1178`
  (also applies to ~1181-1183, ~1372, ~1396)
- Severity: should-fix
- Source: qs-review-coderabbit (Major)
- Scope: pre-existing (security)
- Description: `chargerOptionsHtml`, `personOptionsHtml`, `chargeTime`, and
  `forecastDisplay` come from entity state/attributes and are interpolated
  raw into `innerHTML`. A value containing markup or quotes can break the
  shadow DOM and inject arbitrary nodes.
- Proposed fix: Run each interpolated entity-derived string through the
  inherited `this._escapeHtml(...)` before building the HTML. (Radiator
  already follows this pattern — match it.)

### [should-fix] S4 — escape pool-mode option text before innerHTML (pool)
- File: `custom_components/quiet_solar/ui/resources/qs-pool-card.js:328-329`
- Severity: should-fix
- Source: qs-review-coderabbit (Major)
- Scope: pre-existing (security)
- Description: `o` and `translatePoolMode(o)` are inserted raw into
  `modeOptionsHtml`; a mode label containing quotes/markup can break the
  `<option>` list and inject DOM. Radiator already escapes this pattern.
- Proposed fix: `this._escapeHtml(...)` the option value and label.

### [should-fix] S5 — make in-ring action controls keyboard reachable (car)
- File: `custom_components/quiet_solar/ui/resources/qs-car-card.js:1367-1384`
  (also applies to ~1623-1776)
- Severity: should-fix
- Source: qs-review-coderabbit (Major)
- Scope: pre-existing (a11y) — interacts with AC5 / S16
- Description: The in-ring controls (solar priority, force-now, finish-time)
  render as plain `div`s with only click/touch handlers. Keyboard users
  can't trigger them — and this partially undercuts the AC5 "S16 keyboard
  activation" claim, since the handler is wired but the element isn't
  focusable.
- Proposed fix: Add `role="button"`, `tabindex="0"`, and wire
  `this._registerKeyActivation(...)` for each control.

### [should-fix] S6 — make custom toggle controls focusable (pool)
- File: `custom_components/quiet_solar/ui/resources/qs-pool-card.js:354-355`
  (also applies to ~383, ~418-432)
- Severity: should-fix
- Source: qs-review-coderabbit (Major)
- Scope: pre-existing (a11y) — interacts with AC5 / S16
- Description: The toggles are plain `div`s, but `_wirePowerButton` /
  `_wireGreenButton` add keyboard handlers; without `role="button"` and
  `tabindex="0"` neither can receive focus, so keyboard users can't operate
  the main controls.
- Proposed fix: Add `role="button"` and `tabindex="0"` to each toggle.

### [should-fix] S7 — keep flame RAF alive until engine reaches idle
- File: `custom_components/quiet_solar/ui/resources/qs-radiator-card.js:286-298`
- Severity: should-fix
- Source: qs-review-coderabbit (Major)
- Scope: flame-engine area
- Description: `showAnimation` shuts the RAF loop off as soon as `running`
  flips false, but the flame engine only decays to `STILL_AMP` on subsequent
  ticks. So the grey flame can freeze in a mid-flicker shape after an on→off
  transition.
- Proposed fix: Keep the RAF loop running until the engine reports it has
  reached its idle/still state (e.g. expose an `isIdle()` / check
  `getCurrentAmp()` ≈ `STILL_AMP`), then stop. Verify other cards using the
  flame engine don't regress.

### [should-fix] S8 — derive drag snap range from `max_default_hours`, not hardcoded 12h
- Files:
  - `custom_components/quiet_solar/ui/resources/qs-climate-card.js:1450-1458`
  - `custom_components/quiet_solar/ui/resources/qs-water-boiler-card.js:1317-1326`
  - `custom_components/quiet_solar/ui/resources/qs-radiator-card.js:490-493`
- Severity: should-fix
- Source: qs-review-coderabbit (Major, repeated across 3 cards)
- Scope: pre-existing (functional)
- Description: `allowedHours` is hard-coded to `0..12`, so targets above 12h

- Proposed fix: Derive `allowedHours` from the entity's configured
  `max_default_hours` (fallback 12). Apply consistently to all three cards.
  Consider hoisting the derivation into the shared ring base if it's
  identical across cards.

### [should-fix] S10 — branded colour literals in shared CSS fallback (AC6)
- File: `custom_components/quiet_solar/ui/resources/shared/qs-card-styles.js:20-26`
- Severity: should-fix
- Source: qs-review-acceptance-auditor (AC6)
- Scope: AC6 policy
- Description: `const colors = palette || { primary:'#2196F3',
  gradStart:'#00bcd4', gradEnd:'#8bc34a', animStart:'#00e1ff',
  animEnd:'#0066ff' }` — the gradient/anim values are branded (the
  pre-refactor palette). The file's own header policy and AC6 require zero
  branded colour literals in the shared module. Dead code today (all 6
  callers pass a truthy palette) but contradicts the documented policy and
  no test enforces it.
- Proposed fix: Drop the branded fallback (fall back to neutral / `var(--*)`
  values, or make `palette` a required argument). If a fallback is genuinely
  wanted, amend AC6's policy to permit a documented neutral fallback. Add a
  test asserting no branded hex literals in the shared CSS module. (Consider
  also adding the `#2ecc71` on-state green to the named-anchor list for
  clarity.)

### [should-fix] S14 — don't depend on the real clock in qs_tag resolution test
- File: `tests/test_ui_dashboard.py:862-872`
  (`test_generate_qs_tag_high_resolution_distinct_within_one_second`)
- Severity: should-fix (minor, flaky)
- Source: qs-review-coderabbit (Minor) + qs-review-blind-hunter
- Description: The test asserts `len({_generate_qs_tag() for _ in range(50)})
  > 1`, depending on real `time.time_ns()` resolution. On platforms with
  coarse granularity (some Windows builds) all 50 tight-loop calls could
  collide → flaky.
- Proposed fix: `monkeypatch` `time.time_ns` to return a controlled
  increasing sequence so the test deterministically produces multiple
  distinct tags.

### [nice-to-have] N3 — orphan `.qstmp` leaked on mid-write failure
- File: `custom_components/quiet_solar/ui/dashboard.py:424-427`
- Severity: nice-to-have
- Source: qs-review-edge-case-hunter
