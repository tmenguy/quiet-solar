### [must-fix] Embedded CR/LF in HTTP header value raises uncaught `ValueError`
- File: `scripts/qs/spawn_session.py:166` (`_api` except tuple)
- Severity: must-fix
- Source: qs-review-edge-case-hunter
- Description: When the `x-opencode-directory` header value contains CR/LF (or any other character `http.client` rejects with `ValueError`), `urllib.request.urlopen` raises `ValueError: Invalid header value b'/tmp/wt\\nMalicious: yes'` from inside `http.client`. The current except tuple in `_api` is `(HTTPError, URLError, OSError, http.client.HTTPException, json.JSONDecodeError)` — `ValueError` is not a subclass of any of these, so the exception propagates out of `_api`, out of `spawn_session`, and the script dies with a Python traceback instead of routing through the fallback-CLI path AC #5 promises. Pre-call `.strip()` (added in fix plan #02 should-fix #8) only handles surrounding whitespace; embedded CR/LF survives. Reproduces with `python scripts/qs/spawn_session.py --agent qs-create-plan --directory "$(printf '/tmp/wt\nMalicious: yes')"`.
- Proposed fix: Two complementary defenses (apply both):
  1. **Widen the `_api` except tuple** to include `ValueError`, routing it through `SpawnSessionError` so the standard fallback-CLI path activates. The minimum change is one line:
     ```python
     except (HTTPError, URLError, OSError, HTTPException, JSONDecodeError, ValueError) as exc:
     ```
  2. **Reject control chars upstream** in `main()`. Add a helper `_reject_control_chars(name: str, value: str) -> None` that raises `parser.error` if `value` contains any of `\r`, `\n`, `\x00`, or other ASCII control chars (`< 0x20` except `\t`). Call it for `--directory`, `--agent`, `--title` (if provided), `--prompt`. Layered defense — caller error gets the clearer message; bug-class regressions still get caught by the widened except.
- Tests:
  - `test_invalid_header_value_falls_back` — monkeypatch `urllib.request.urlopen` to raise `ValueError("invalid header")`; assert `fallback_cli` (or `fallback_unavailable`) and the standard JSON shape.
  - `test_control_chars_in_directory_rejected` — parametrized over `["/tmp/wt\n", "/tmp/wt\r", "/tmp/wt\x00", "/tmp\nwt"]`; assert argparse exit 2 with a clear `parser.error` message.
  - Same parametrization for `--agent`, `--prompt`, and `--title`.

### [should-fix] `session_id` URL escape doesn't neutralize `..` traversal segments
- File: `scripts/qs/spawn_session.py` (around the `safe_id = urllib.parse.quote(session_id_str, safe="")` line)
- Severity: should-fix
- Source: qs-review-blind-hunter
- Description: `urllib.parse.quote(session_id_str, safe="")` escapes `/` to `%2F` but per RFC 3986 the unreserved set `[A-Za-z0-9_.-~]` is never escaped — so `..` literally passes through. The current `test_spawn_session_url_escapes_session_id` only asserts `/` → `%2F`; an id of `"abc..delete-me"` survives untouched in the URL path. If any HTTP middleware (corporate proxy, reverse proxy, container ingress) does URL-path normalization before forwarding to the OpenCode server, the request could still resolve to a traversal-rewritten path.
- Proposed fix: Two-layer defense:
  1. **Validate the id at parse time.** After `safe_id` is built, reject any id whose original string contains `..` as a substring or as a path segment:
     ```python
     if ".." in session_id_str:
         raise SpawnSessionError(
             f"server returned an id with traversal segments: {session_id_str!r}",
             ...
         )
     ```
  2. **Pin the test.** Extend `test_spawn_session_url_escapes_session_id` (or add a sibling `test_spawn_session_rejects_traversal_id`) parametrized over `["abc/../delete-me", "..", "../etc/passwd", "abc..def"]`. The first three should route through fallback; the last (`abc..def`, which contains `..` but isn't a traversal pattern per se) is also rejected for safety — consistent contract.

### [should-fix] Orphan temp file leak in `_opencode_cli_command` on mid-stream `f.write` failure
- File: `scripts/qs/launchers/opencode.py:219` (`_opencode_cli_command`, the `NamedTemporaryFile` block)
- Severity: should-fix
- Source: qs-review-edge-case-hunter
- Description: The current flow is:
  ```python
  with tempfile.NamedTemporaryFile(..., delete=False) as f:
      f.write(...)  # ← raises here
      script_path = Path(f.name)  # ← never assigned
  ```
  If `f.write` raises (disk full, quota exceeded, broken pipe), the `NamedTemporaryFile(delete=False)` has already created the file on disk but `script_path` is still `None`. The `except OSError: if script_path is not None: script_path.unlink(...)` branch then does nothing, and the partial file is leaked.
- Proposed fix: Capture the path BEFORE writing:
  ```python
  with tempfile.NamedTemporaryFile(...) as f:
      script_path = Path(f.name)
      f.write(...)
  ```
  Now the cleanup branch always has a non-None `script_path` when an exception fires inside the `with`. Add `test_opencode_cli_command_cleans_temp_file_on_write_failure` — monkeypatch `NamedTemporaryFile`'s file-like to raise `OSError` from `.write`; assert the temp file is unlinked.

### [should-fix] `build_existing_session_prompt` emits ugly `../../../` path with empty `work_dir`
- File: `scripts/qs/launchers/phases.py` (`build_existing_session_prompt`)
- Severity: should-fix
- Source: qs-review-edge-case-hunter
- Description: The guard at the top of the helper validates `fix_plan_path` and `pr_number` but not `work_dir`. When `work_dir == ""` (or `"   "`), `os.path.relpath("/abs/path/to/fix.md", "")` returns `"../../../../../../abs/path/to/fix.md"` (relative from the CWD up to root, then back down). The user-visible prompt embeds this artifact.
- Proposed fix: Short-circuit when `work_dir` is empty / whitespace — fall back to the verbatim absolute `fix_plan_path` like the existing `ValueError` branch does:
  ```python
  if not work_dir or not work_dir.strip():
      fix_plan_rel = fix_plan_path
  elif os.path.isabs(fix_plan_path):
      try:
          fix_plan_rel = os.path.relpath(fix_plan_path, work_dir)
      except ValueError:
          fix_plan_rel = fix_plan_path
  else:
      fix_plan_rel = fix_plan_path
  ```
  Add `test_build_existing_session_prompt_handles_empty_work_dir` parametrized over `["", "   ", "\t"]` and assert the prompt contains the verbatim absolute fix_plan_path (not the traversal artifact).
