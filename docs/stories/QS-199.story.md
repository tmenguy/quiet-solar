## Scope (single paragraph)

Extract a base class + one sub-base + composition helpers from the six
`ui/resources/qs-*-card.js` files (~9,400 LOC total) into five shared
modules under `ui/resources/shared/`. Modify
`ui/dashboard.py:_async_copy_and_register_resources` to (a) skip
Lovelace registration for files in subdirectories of `_RESOURCES_DIR`,
and (b) rewrite `from './shared/*.js'` import URLs at copy time to
inject the same `?qs_tag=<EPOCH>` cache-buster used for top-level
cards. Each card migrates to extend the appropriate base and consume
the engines. Particle systems (boiler bubbles/steam/glow, climate
snow/wind, car sparkles/lightning, pool temp-tint) **stay per-card**
— see "Why particle systems stay per-card" below. Defensive hardening
is uplifted only where it falls out from extracting a single shared
implementation; no broader uplift. The refactor is **invisible to end
users** modulo a tightly-bounded palette-drift envelope (see "Allowed
palette drift" below).

## Allowed palette drift

The user explicitly authorised "stuff that may be now better aligned,
more commonality in shapes and some color that may be slightly off
before". This story bounds that envelope:

- **Allowed**: normalising hex literals to 6 digits (e.g. `#FFF` →
  `#FFFFFF`), consolidating two literals that differ by ≤ 1 in any
  byte (e.g. `#FF5722` and `#FF5723`) to a single value, collapsing
  whitespace differences in CSS strings.
- **Forbidden**: any change to hue/saturation/lightness perceptible
  by eye, any change to the per-card warm/cool palette identity
  (radiator stays orange, climate-cool stays blue, car charging stays
  cyan, etc.).
- **Verified by**: Phase F visual smoke (no automated bound — eyeball).

## Why particle systems stay per-card

Each particle system (boiler bubbles, boiler steam, climate snow,
climate wind, car sparkles, car lightning, pool temp-tint) is:

- Used by **exactly one card**;
- Has **5+ rounds of review-fix tuning** baked into its constants and
  spawn/advance/retire algorithm (see QS-200/QS-211/QS-214/QS-216/
  QS-232/QS-210 story files);
- Shares ~20 lines of generic lifecycle scaffolding with the others
  (spawn-gate / RAF tick / opacity update / teardown) but those
  ~20 lines diverge in lerp envelopes, spawn predicates, retire
  conditions, and geometry-aware fades.

Extracting a generic `QsParticleEngine` would gain <120 LOC of
deduplication at very high regression risk (every existing tuning
hint would need to ride into the engine as a constructor arg). The
duplication payoff doesn't justify the risk. This decision is
explicit and documented; future readers should not redo the analysis
without new evidence.

### `QsCardBase` (`shared/qs-card-base.js`)

Exports `class QsCardBase extends HTMLElement` and the geometry pure
functions as named exports (`deg2rad`, `rad2deg`, `polar`, `arcPath`,
`pctToDeg`). Class API:

- **Lifecycle**: `setConfig(config)`, `connectedCallback()`,
  `disconnectedCallback()` (resets `_isInteractingMode`,
  `_isInteractingTarget`, `_isProcessingModeChange`, `_modalOpen`),
  `set hass(hass)` (gated by interaction flags), abstract `_render()`
  throws.
- **`getCardSize()` is NOT defined on the base** — each card overrides
  (car returns `6`, others return `5`). This is a deliberate per-card
  override point (concrete-planner finding #2).
- **Service callers**: `_entity(id)`, `_call(domain, service, data)`,
  `_press`, `_turnOn`, `_turnOff`, `_select`, `_setNumber`, `_setTime`.
- **Defensive utilities** (S6/S8/N12/N13/S16 — see glossary):
  `_escapeHtml(s)`, `_safeNumber(sensor, defaultValue)` with
  trim + `Number.isFinite` guard (climate-card's hardened version),
  `_fmt(num, round)`, `_isValidState(state)`, `_formatTime(timeStr)`,
  `_parseTimeToMinutes(txt)`, `_formatHm(mins)`.
- **`_showDialog(opts)` — precise API** (per concrete-planner
  finding #10): `opts = {title, message, customContent?, buttons,
  onClose?}`. `buttons` is array of `{text, variant?, onClick?}`. If
  empty, base appends a "Close" fallback (N12). Each rendered button
  gets `_registerKeyActivation` wired automatically. Base sets
  `this._modalOpen = true` on entry and resets it inside a
  `try/finally` after `onClick` (N13). Returns the `wrap` element
  for caller customisation (e.g., reading dialog selects in onClick).
- **Keyboard activation (S16)**: `_registerKeyActivation(el, action)`
  wires `Enter`/`Space` on a `role="button" tabindex="0"` element.
- **Per-instance ID counter**: `_instanceId(prefix)` returns
  `<prefix>-<n>` where `n` is unique per subclass (uses
  `this.constructor._nextInstanceId` so each card-type has its own
  monotonic counter).
- **Dashed-arc RAF loop**: `_startArcDashAnimation()` /
  `_stopArcDashAnimation()` for the `running_anim` SVG path with
  stroke-dashoffset animation. RAF gating left to the card.
- **Wire-helpers** (hoisted from the duration sub-base because **car
  uses them too** — concrete-planner finding #1):
  - `_wireTargetHandle({getRingSvg, getHandle, center, ringCirc,
    startDeg, endDeg, rangeDeg, hoursToPct, pctToHours, allowedHours,
    entityId, onCommit})` — pointer + mouse + touch with
    `PointerEvent` capability check; snap-to-allowedHours; S17
    try/finally around `_setNumber` so drag-release guards always
    clear.
  - `_wireTimePicker({buttonEl, entityId, currentMins, localStateKey})`
    — opens the hour/minute picker dialog via `_showDialog`; S9
    5-second local-state clear timer.
  - `_wireResetButton({buttonEl, entityId})` — confirmation dialog.
  - `_wirePowerButton({buttonEl, swEntity, entityId})`.
  - `_wireGreenButton({buttonEl, swEntity, entityId})`.

### `QsRingDurationCardBase` (`shared/qs-ring-duration-base.js`)

`class QsRingDurationCardBase extends QsCardBase`. Used by
on-off-duration, radiator, pool, water-boiler, climate. **Not** used
by car (car has a different full-circle ring + 3 inside-disc buttons
instead of one override-btn). Adds:

- `_buildRingHTML({palette, ringCirc, gapDeg, hasGreenBtn,
  hasPowerBtn, hasOverrideBtn, canDragHandle, ...})` — returns the
  outer `<div class="ring">` SVG markup: linear-gradient defs,
  background-arc + progress-arc, override-btn cover circle (QS-217),
  target-handle circle + text, power-btn + green-btn + override-btn
  DOM elements. Cards layer their backdrop-specific SVG inside the
  returned template.
- `_wireOverrideButton({buttonEl, entityId, overrideBtnClickable})` —
  confirmation dialog + state-aware class assignment (active /
  resetting / disabled).
- `_wireBistateMode({selectEl, entityId, translationNamespace})` —
  focus/blur/change handlers; M2 try/finally + 300ms setTimeout
  cleanup; `showPicker()` fallback to `focus()` on pill-click.

### `qs-card-styles.js`

Exports `baseCardCSS(palette, options)` returning the shared CSS
template-literal string. Includes: `:host` custom properties
(`--pad`, `--ring-text-shadow`), `.card`, `.card.off-grid`,
`.card.disabled`, `.card-title`, `.top`, `.below`, `.below-line`,
`.pill`, `.hero`, `.ring`, `.ring svg`, `.ring .power-btn`,
`.ring .green-btn`, `.ring .override-btn`, `.ring .center`,
`.ring .target-block`, `.ring .target-label/.target-value`,
`.ring .from-to-row`, `.ring .stack`, `.time-btn`, `.modal`,
`.dialog`, `.btn`, `.btn.secondary/.primary/.danger`, `select`,
`button`, `button.pill`, `button.outline`, `.danger`.

**Colour literal policy** (per plan-critic finding CL3): the shared
CSS allows neutral CSS literals (`rgba(0,0,0,...)`,
`rgba(255,255,255,...)`, `var(--divider-color)`, theme variables) but
**no branded colour literals** (no orange/blue/green hexes or hsla).
Branded palette flows through `palette` argument as
`{primary, gradStart, gradEnd, animStart, animEnd}` and per-state
fixed colours (`#2196F3` power-blue, `#4CAF50` solar-green,
`#FF9800` override-orange) remain hardcoded in the shared CSS
because they are **semantic anchors** (not card-branded), already
identical across all 6 cards today.

Cards append their own card-specific CSS (e.g. car's `.sun-btn`/
`.rabbit-btn`/`.time-btn` positioning, water-boiler's
`.temp-row`, climate's nothing-card-specific).

### `qs-anim-flame.js`

Exports:

- `FLAME_CONSTANTS = { FLAME_WIDTH: 480, FLAME_BOTTOM_Y: 400,
  STILL_AMP: 0, DANCE_AMP: 8, STATIC_PEAK_HEIGHT: 30, LERP_RATE: 2,
  LERP_DT_CEIL: 0.1, AMP_REGEN_THRESHOLD: 0.25,

  FLAME_BASE_MIN_PCT: 0.2, FLAME_BASE_MAX_PCT: 0.8 }`
- `class QsFlameEngine` — constructor signature (per concrete-planner
  finding R5): `({ layerTeethCounts, layerTipFlickerHz,
  layerBaseHeights, layerTipAmpMults, runningFills, greyFills })`.
  Both radiator and climate supply the same numeric values today
  (`[3,4,5]`, `[8,7,9]`, `[150,120,90]`, `[1.2,1.0,0.8]`,
  `FLAME_FILLS`, `FLAME_GREY_FILLS`) — but each owns its constants in
  the card file (per AC6). Methods: `step(dt, fireOn) → {shouldRegen}`,
  `generatePaths(baseY) → string[layerCount]`, `invalidate()`,
  `reset()`. Engine owns `_currentFlameAmp`, `_tipPhases[]`,
  `_lastFlameAmp`, `_lastFlameBaseY`, `_lastRegenTs`,
  `_needsFlamePrime`. Card owns the DOM refs and the RAF outer loop;

### `qs-anim-wave.js`

Per concrete-planner finding R4 — **no class**, only constants +
pure helper:

- `WAVE_CONSTANTS = { WAVE_WIDTH: 480, WAVE_BOTTOM_Y: 400,
  LAYER_SCROLL_OFFSET: 1.2, LAYER_PHASE_OFFSET: 2.1, LERP_RATE: 2,
  LERP_DT_CEIL: 0.1, AMP_REGEN_THRESHOLD: 0.25,
  LEVEL_REGEN_THRESHOLD: 0.01, PHASE_WRAP: 1e6, PHASE_TO_PX: 60 }`
- `generateWavePath(width, amplitude, frequency, phase, yOffset)` —
  pure function (~15 LOC), used by pool (3 layers), water-boiler
  (3 layers), climate-cool (3 layers), car (1 layer). Each card's RAF
  step body lives in the card and operates on its own
  `_currentAmplitude` / `_currentSpeed` / `_wavePhase`. Boilerplate
  duplication here is ~30 LOC × 4 = ~120 LOC; an engine class would
  add comparable LOC of feature-flagging (layerCount, palette shape,
  optional bubble/steam/snow/sparkle hooks). Net savings near zero;
  keeping it as constants + helper is the right granularity.

### Inheritance hierarchy

- `QsCardBase extends HTMLElement` — used directly by: **car**.
- `QsRingDurationCardBase extends QsCardBase` — used by: **pool,
  on-off-duration, radiator, water-boiler, climate**.

Decision rule going forward (per plan-critic finding I4): **lifecycle
+ service callers + helpers that mutate `this.*` flags → inheritance;
pure pipeline transforms (engines, geometry, CSS templates) →
composition**.

### Python-side (`ui/dashboard.py`)

Modify `_async_copy_and_register_resources`:

1. Walk source tree recursively (existing behaviour preserved).
2. **A1a — subdir-skip for Lovelace registration**: only call
   `_async_update_resource` for files where the relative path under
   `_RESOURCES_DIR` has no parent directory (i.e., top-level only).
   Files in `shared/` are still copied to `www/quiet_solar/shared/`
   but are NOT registered as Lovelace resources.
3. **A1b — import-URL rewrite**: on every `.js` copy, the file
   content is regex-rewritten before the write:

   ```python
   IMPORT_REWRITE_RE = re.compile(
       r"""from\s+(['"])(\./shared/[^'"?]+\.js)(\?[^'"]*)?\1"""
   )
   # Always overwrite the query string — never skip (plan-critic R1):
   content = IMPORT_REWRITE_RE.sub(
       rf"from \1\2?qs_tag={tag}\1", content
   )
   ```

   Same `tag` value flows from the single `_generate_qs_tag()` call
   inside `async_update_resources` to both (a) the Lovelace
   registration URL and (b) the rewritten in-file import URLs
   (plan-critic R2).

4. **Logging contract** (dev-proxy finding I13): any new `_LOGGER`
   call uses lazy `%s`, no f-strings, no trailing period — e.g.,
   `_LOGGER.debug("Skipped lovelace registration for shared module %s", relpath)`.

## Defensive-hardening glossary

(Inline-defined per plan-critic finding CL2 / dev-proxy clarify-4.
These codes appear in inline comments in the card source files; this
glossary is the canonical reference.)

| Code | Name | One-line definition |
|---|---|---|
| S6 | escape-HTML | Defensive `_escapeHtml(s)` on user-/3rd-party-controlled strings interpolated into `innerHTML` (title, dialog message, unit). |
| S7 | disconnect-flag-reset | `disconnectedCallback` clears every `_isInteracting*` / `_modalOpen` flag so a re-attach mid-interaction doesn't lock `set hass`. |
| S8 | safe-number | `_safeNumber(sensor, default)` filters `''`/`unknown`/`unavailable` BEFORE `Number(...)`, plus `Number.isFinite` guard against `±Infinity`. |
| S9 | local-state-cleanup | Apply handlers that set `_localXxx` also schedule a 5-second clear timer so out-of-band backend updates aren't masked indefinitely. |
| S14 | NaN-safe state reads | Use `_safeNumber` (not `Number(s?.state \|\| N)`) for every sensor-state-to-number conversion that feeds an SVG attribute. |

## Rollback

(Per plan-critic finding I1.)

Single revert path: `git revert <merge-commit>` reverts all card
migrations + the dashboard.py changes + the new shared modules + the
test updates + the doc update in one go. After revert, **end users
should perform a browser hard-refresh** (Ctrl-Shift-R / Cmd-Shift-R)
once because the rewritten import URLs cached in their browser will
404 against the reverted (pre-refactor) cards. The
`?qs_tag=<EPOCH>` cache-buster on the top-level Lovelace URLs makes
this self-healing on the next HA restart, but a manual refresh is
faster.

No data migration involved (cards are stateless). No HA config
changes. No user-visible content change.
