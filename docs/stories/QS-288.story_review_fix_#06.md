  (three-mirror co-edit rule applies)
- Severity: should-fix
- Source: qs-review-edge-case-hunter
- Description: the K3 rule's trigger "markdown-only change sets"
  under-covers the vacuousness it exists to close — testmon is blind
  to ALL non-Python files, and the blindness is per-file, not

  `.claude/settings.json` — pinned by

  so the rule stays silent while `--impacted` selects 0 tests; (b) a
  mixed change set (agent `.md` + any `scripts/qs/*.py` edit) makes
  the condition false, `--impacted` selects only the py-side tests,
  and a broken pinned marker in the md surfaces only in CI.
- Proposed fix: reword the trigger in all four places, identically:
  "For change sets touching any `tests/qs`-pinned non-Python file
  (agent files, commands, workflow docs, `.claude/settings.json`) —
  even when Python files changed too — also run
  `python scripts/qs/quality_gate.py --quick tests/qs` before commit
  (testmon cannot see non-Python files, so `--impacted` alone is
  blind there)."

  byte-identity and will re-check).
