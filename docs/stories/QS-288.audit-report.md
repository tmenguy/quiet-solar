## §1 Executive summary

**Speed.** The mechanical layer is bimodal: everything except pytest is
fast (`next_step.py` 0.05s, `check_doc_drift.py` 0.07s, cache hit
0.12s, dev-only scope skip 0.12s), while anything that boots pytest
pays a large fixed cost — a **complete no-op `--impacted` run costs
10.2s** (0/6339 tests selected) and a 43-test `--quick` run costs 7.4s.
Stage attribution: ~0.7s git plumbing, **~9.7s pytest boot/collection
(two full boots: a 3.07s `--collect-only` subprocess that exists only
to compute a progress denominator, plus the `--testmon` run that
re-collects all 6339 tests per xdist worker)**, ~0.2s diff-cover.
`context.py` costs ~1.0s per agent startup, ~0.85s of which is two
serial `gh` network calls. The full whole-repo gate reference is
**1129s (18m49s)** locally.

**Top 3 speed wins by score** (formula: seconds saved × events/week ÷
effort weight; all arithmetic in §4):

1. **S-2 — CI full-gate sharding** (score ≈ 200, assumption-sensitive):
   shard the CI pytest job; saves ~15 min per blocking CI wait.
2. **S-3 — trim agent-startup mandated reading** (score ≈ 125,
   estimate): ~56 KB of mandated reads per orchestrator session is
   LLM-latency, not mechanical.
3. **S-1 — kill the gate's fixed pytest overhead** (score ≈ 69, best
   measured): skip the collect-only denominator pass, single-process
   fast path for tiny selections. The most robustly evidenced win.

**Discrepancies.** 22 findings (D-1…D-22). Headlines: the harness
contract is **self-contradictory** — `harness.md` declares agent bodies
"identical" while `project-rules.md` says harness-specific sections
"legitimately differ", and reality has six distinct delta classes
(D-1); the opencode mirrors received **four deliberate protocol
improvements never back-ported** to claude/cursor (D-2); both
implement-phase slash commands still tell the fallback path to run the
**bare full gate**, contradicting the QS-276 `--impacted` invariant
(D-3); issue #288's mislabel was **not** setup-task's doing — it was
`issue-triage.yml`'s substring matcher (`'ev'` ⊂ "dev", `'ui'` ⊂
"quiet"/"build") with no dev-infra label in the taxonomy (D-10); and
two `docs/agents/` concept docs are materially wrong against source
(D-11).

**Dispositions** are proposals; the user decides (per story). Inline
candidates are pure text/doc fixes; anything semantic or
behavior-touching is proposed as a follow-up issue.

## §4 Speed findings (sorted by score)

Ranking model: `score = seconds saved × events/week ÷ effort weight`
(S=1, M=3, L=9). **Frequency assumptions** (stated once, used
throughout): ~3 tasks/week; ~25 orchestrator-session startups/week;
~40 `--quick` TDD iterations/week; ~12 `--impacted` runs/week of which
~5 on docs-only trees; ~3 merges/week; ~2 blocking CI waits/week.

**S-2 · CI full-gate sharding · score ≈ 200 (assumption-sensitive).**
Cost: local full gate = 1129s wall / 3236s user (§2); on 4-core CI
runners the same suite is plausibly 20–30 min (`user/4 ≈ 13.5 min`
lower bound; not measured here — CI-side). Proposal: shard CI pytest
(e.g. 4-way matrix via `pytest-split` or xdist groups), keeping
`--cov-fail-under=100` via combined coverage. Saving: ~900s per
*blocking* wait. Arithmetic: `900 × 2 ÷ 9 = 200`. Effort L.
Assumption-sensitive: if the user rarely blocks on CI, score drops
proportionally (1 wait/wk → 100). **Proposed: follow-up.**

**S-3 · Trim agent-startup mandated reading · score ≈ 125 (estimate,
LLM-side).** Cost: every orchestrator session mandates
`project-rules.md` (10.1 KB) + often `project-context.md` (13.7 KB) +
story (QS-288's: 20.5 KB) + agent body (1.3–9.6 KB) + `CLAUDE.md`
(2.8 KB) ≈ up to ~56 KB ≈ 14K tokens before work starts — LLM
processing latency, not mechanical. Estimate ~15s per startup.
Arithmetic: `15 × 25 ÷ 3 = 125`. Proposal: deduplicate
`CLAUDE.md`↔`project-rules.md` overlap (commands block appears in
both), split `project-context.md`'s 42 rules into phase-relevant
subsets, stop mandating re-reads the phase doesn't need. Effort M.
Honesty: unmeasured; token-time estimate. **Proposed: follow-up.**

**S-1 · Kill the gate's fixed pytest overhead · score ≈ 69 (best
measured).** Cost: 43-test `--quick` = 7.37s; 0-test `--impacted`
pytest phase = 9.7s; ~33s CPU each — dominated by per-worker
boot+collection of 6339 tests under the full plugin stack (HA imports
in `tests/conftest.py`). Proposals: (a) drop the `--collect-only`
denominator subprocess in `--impacted` (see S-5 — it's a strict subset
of this finding); (b) single-process fast path when the target/selected
set is small (skip `-n auto` worker spin-up for <~50 tests); (c) audit
auto-loaded plugins for the quick path. Saving: est. 4s ×
(40 quick + 12 impacted) = 208s/wk. Arithmetic: `4 × 52 ÷ 3 = 69.3`.
Effort M. **Proposed: follow-up.**

**S-4 · `--impacted` docs-only early-exit · score ≈ 17.** Cost: a
docs/markdown-only diff still pays the full 10.2s (§2 — 0 tests
selected, diff-cover vacuous). No early exit exists:
`check_impacted` (`quality_gate.py:1388-1478`) never inspects the
changed-file list ( `--impacted` short-circuits `main()` at `:1644`
before `_detect_scope`). Natural seat identified: after base resolve
(`:1428`), one `git diff --name-only` — if no `.py` changed, PASS
immediately. Saving: 10.1s × 5/wk = 50.5s/wk. Arithmetic:
`10.1 × 5 ÷ 3 = 16.8`. Effort M (gate logic + tests). **Proposed:
follow-up** (same issue as S-1/S-5/S-7/D-18).

**S-5 · Skip the `--collect-only` denominator pass · score ≈ 12
(subset of S-1).** Evidence: `quality_gate.py:545-563,1343` — a full
6339-test collection subprocess (3.07s measured) exists only to
compute `milestone_every` for progress printing. Saving: 3.1s ×
12/wk = 37s/wk. Arithmetic: `3.1 × 12 ÷ 3 = 12.3`. Effort M (bundled).
**Proposed: follow-up.**

**S-6 · Parallelize/merge `context.py`'s gh calls · score ≈ 10.5.**
Evidence: two serial calls (`context.py:44,60` →
`utils.find_pr_for_branch`), measured 0.44s + 0.40s of the 0.98s
total. Proposal: run both in a thread pair, or one GraphQL query;
optional per-issue title cache. Saving: ~0.42s × 25/wk. Arithmetic:
`0.42 × 25 ÷ 1 = 10.5`. Effort S. **Proposed: follow-up.**

**S-7 · TTL-cache the `--impacted` `git fetch` · score ≈ 6.**
Evidence: `quality_gate.py:1020` fetches `origin main` on every run
(15s cap; ~0.5s measured locally, worse on slow networks). Proposal:
skip the fetch if `.git/FETCH_HEAD` is fresher than ~10 min.
Saving: 0.5s × 12/wk. Arithmetic: `0.5 × 12 ÷ 1 = 6`. Effort S.
**Proposed: follow-up** (bundled with S-1/S-4/S-5).

**S-8 · `worktree-setup.sh` cache seeding · score ≈ 5.** Evidence:
`scripts/worktree-setup.sh:340-371` `cp -R` of `.mypy_cache` +
`.testmondata` (3.7 MB+) incl. rm-and-recopy on rerun; plus up to 4
network ops (`:256,415,440,469,535`). Saving: est. 5s × 3
setups/wk. Arithmetic: `5 × 3 ÷ 3 = 5`. Effort M. **Proposed:
wontfix** for now (setup is once-per-task; low value).

**S-9 · finish-task double `gh pr view` (claude/cursor) · score ≈
1.3.** Evidence: D-2.3 — opencode already merged the calls. Saving:
0.44s × 3 merges/wk. Arithmetic: `0.44 × 3 ÷ 1 = 1.3`. Effort S.
**Proposed: follow-up** (rides along with the D-2 re-sync; zero
marginal cost).

**S-10 · Instrumented end-to-end phase profiling · REJECTED (explicit
AC6 entry).** Rationale: the scratch timestamped wrapper + the

wall time (0.69 / 9.7 / 0.16 split, §2), C3 attributes `context.py`
to the two `gh` calls within 0.05s, and C1's static walk covers the
phase protocols; per-stage instrumentation inside the gate would add

land and the residual is still surprising, revisit then.
