## Findings to address

### must-fix

**MF-1 (`ls-remote` matches tags too, blind-hunter + edge-case-hunter):**
at `scripts/worktree-setup.sh:191`, `git ls-remote origin "$BRANCH"`
matches refs in both `refs/heads/` and `refs/tags/`. If a tag named
`QS_<N>` ever exists, `awk '{print $1}'` emits multiple SHAs, `REMOTE_TIP`
becomes multi-line, the `!=` compare is unconditionally true, and
`merge-base --is-ancestor` errors on the multi-arg form. Parent story's
AC-2 verification already uses `--heads`; this is inconsistent.

**MF-2 (untracked-directory `rm -rf` skips dirty/stash check,
edge-case-hunter):** at `scripts/worktree-setup.sh:95-99`, when
`IS_TRACKED_WORKTREE="no"`, the script `rm -rf`'s `$WORKTREE_DIR` with
no cleanliness check. Fix #02's SF-2 aimed to prevent silent data loss
during destructive recovery; the symmetric arm bypasses that protection.

### should-fix

**SF-1 (`merge-base --is-ancestor` against unfetched SHA, blind-hunter):**
at `scripts/worktree-setup.sh:195-198`, if `REMOTE_TIP` is a SHA the
local worktree hasn't fetched yet, `merge-base` errors (not "not
ancestor"); the `!` inversion misclassifies as divergence.

**SF-2 (detached-HEAD silently triggers recovery, blind-hunter):** at
`scripts/worktree-setup.sh:57-58`, `rev-parse --abbrev-ref HEAD` returns
the literal string `HEAD` in detached state — equality check fails
silently and falls through to destructive recovery.

**SF-3 (push under `set -e` with no actionable message, edge-case-hunter):**
at `scripts/worktree-setup.sh:217`, push failure (auth expired, network
drop, pre-push hook, credential prompt hang) bubbles up bare. Fix #02
added explicit `||` handling for `worktree remove --force`; the same
pattern is missing for the push itself.

**SF-4 (dirty/stash check swallows command failures, edge-case-hunter):**
at `scripts/worktree-setup.sh:71-74`, `2>/dev/null || true` masks
corrupt-index / broken-`.git` errors. If `git status` actually fails,
`DIRTY` comes back empty → recovery treats worktree as clean.

**SF-5 (`WORKTREES_DIR` canonicalization mismatch on macOS,
edge-case-hunter):** at `scripts/worktree-setup.sh:30-36`, `pwd -P`
canonicalizes symlinks (`/var` → `/private/var` on macOS). `git worktree
add` records the path verbatim. If git's records diverge from `pwd -P`,
`grep -qxF` (line 53) never matches, `IS_TRACKED_WORKTREE` stays "no",
and `rm -rf` runs against a live tracked worktree.

**SF-6 (stale `2>/dev/null` reference in Adversarial Review Notes,
coderabbit):** at `docs/stories/QS-173.story.md:322`, the prose mentions
`"Trade-off of `2>/dev/null` in Task 2 acknowledged inline"`, but Task 2
contains no such trade-off. Stale from earlier draft.

### Task B — Hoist dirty/stash check (`scripts/worktree-setup.sh`)

**B.1 (MF-2)** — Move the dirty/stash check block (lines 69-82) from
inside the `IS_TRACKED_WORKTREE="yes"` branch to **above** the
`IS_TRACKED_WORKTREE` switch, so both the tracked-removal and `rm -rf`
arms are guarded. The check needs to handle both:
- Tracked worktree: a real `.git` file/dir is present, `git status` and
  `git stash list` work.
- Untracked stale directory: may or may not have a `.git`, may just
  contain user-created files.

For the untracked case, fall back to checking whether the directory has
**any** non-empty content other than `.git*` — if it has files, refuse.
Suggested form:

```bash
DIRTY=""
STASHES=""
HAS_NONGIT_CONTENT=""
if [ -d "${WORKTREE_DIR}/.git" ] || [ -f "${WORKTREE_DIR}/.git" ]; then
    if ! DIRTY="$(git -C "$WORKTREE_DIR" status --porcelain 2>/dev/null)"; then
        echo "Error: 'git status' failed in ${WORKTREE_DIR}."
        echo "The worktree's git metadata may be corrupted. Resolve manually:"
        echo "  cd ${WORKTREE_DIR} && git status   # inspect error"
        exit 1
    fi
    if ! STASHES="$(git -C "$WORKTREE_DIR" stash list 2>/dev/null)"; then
        STASHES=""  # stash list is repo-wide; failure means git command itself broke
    fi
fi
# Even without a .git, the directory may contain user files.
# Detect any visible content (excluding . and ..).
if [ -d "$WORKTREE_DIR" ]; then
    if find "$WORKTREE_DIR" -mindepth 1 -maxdepth 1 ! -name '.git' ! -name '.git*' -print -quit 2>/dev/null | grep -q .; then
        HAS_NONGIT_CONTENT="yes"
    fi
fi
if [ -n "$DIRTY" ] || [ -n "$STASHES" ] || \
   { [ -z "$DIRTY" ] && [ -z "$STASHES" ] && [ "$IS_TRACKED_WORKTREE" = "no" ] && [ -n "$HAS_NONGIT_CONTENT" ]; }; then
    echo "Error: ${WORKTREE_DIR} has uncommitted changes, stashes, or non-git content."
    echo "  Status:  ${DIRTY:-(none)}"
    echo "  Stashes: ${STASHES:-(none)}"
    echo "  Other:   ${HAS_NONGIT_CONTENT:-(none)}"
    echo "Refusing to destructively recover. Resolve manually:"
    echo "  ls -la ${WORKTREE_DIR}"
    exit 1
fi
```

Note this also lands **SF-4** (distinguish command failure from empty
output — the `if !` form rejects `git status` errors rather than
swallowing them via `|| true`).

Place this block right after the `IS_TRACKED_WORKTREE` detection (after
line 55) and before the `if [ "$IS_TRACKED_WORKTREE" = "yes" ]` switch
at line 56. The downstream branches at lines 56-99 then need only the
destructive operations.

### Task E — Push error wrapper (`scripts/worktree-setup.sh`)

**E.1 (SF-3)** — At line 217, wrap the push with an explicit failure

```bash
echo "Publishing ${BRANCH} to origin and setting upstream..."
if ! git -C "$WORKTREE_DIR" push -u origin "${BRANCH}"; then
```

### Task F — Canonicalize via `git worktree list` (`scripts/worktree-setup.sh`)

**F.1 (SF-5)** — The mismatch arises because `pwd -P` canonicalizes
symlinks but `git worktree add` records the path verbatim. The cleanest
fix is to **not** canonicalize `WORKTREES_DIR_RAW` via `pwd -P`, and
instead match against `git worktree list` using a tolerant comparison
that accepts both the canonical and non-canonical forms.

Replace the canonicalization at lines 30-37 with the raw form, but
*also* compute the canonical form for the membership check:

```bash
WORKTREES_DIR_RAW="${MAIN_DIR}/../${MAIN_BASENAME}-worktrees"
mkdir -p "$WORKTREES_DIR_RAW"
# Use the verbatim form for git operations (git records this exact path
# in `worktree list --porcelain`). Also compute the canonical form so
# membership checks accept either representation under macOS symlinks
# like /var → /private/var.
WORKTREES_DIR="$WORKTREES_DIR_RAW"
WORKTREES_DIR_CANON="$(cd "$WORKTREES_DIR_RAW" && pwd -P)"
WORKTREE_DIR="${WORKTREES_DIR}/${BRANCH}"
WORKTREE_DIR_CANON="${WORKTREES_DIR_CANON}/${BRANCH}"
```

Then at line 52-53 (the `grep -qxF` membership check), accept either:

```bash
if git -C "$MAIN_DIR" worktree list --porcelain \
    | grep -E "^worktree " \
    | sed 's/^worktree //' \
    | grep -qxF -e "$WORKTREE_DIR" -e "$WORKTREE_DIR_CANON"; then
    IS_TRACKED_WORKTREE="yes"
fi
```

(Multiple `-e` patterns to `grep -F` are portable on macOS bash 3.2 and
Linux.)

Rationale: matching either form is cheaper and more robust than trying
to canonicalize on both sides — git's own canonicalization rules vary
across versions and platforms.
