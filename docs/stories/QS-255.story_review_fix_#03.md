### [must-fix] Rework: latch-free contradiction dedup + manual upgrade of an ongoing auto episode
- File: `custom_components/quiet_solar/ha_model/car.py` (attribute :246, `_exit_stale_mode` :824, `check_charger_assignment_contradiction` :849-941, `clear_inferred_flags` :943-947)
- Severity: must-fix
- Source: qs-review-edge-case-hunter (manual-upgrade gap) + user decision (latch removal)
- Description: Two problems with the fix-#02 latch design:
  1. **Manual upgrade blocked** — when an auto contradiction holds the car
     stale (inferred flags deliberately False per D1), a subsequent explicit
     user assignment to that same charger calls the check with `manual=True`
     but hits the already-stale early return (car.py:883) first, so the
     inferred-flag upgrade never applies. The user's ground-truth
     confirmation — doing exactly what the auto notification asked — is
     silently dropped. Same root cause via the charger-side entry point
     (`user_set_selected_car_by_name`, charger.py:3047).
  2. **Unnecessary new state** — `_contradiction_latched_charger_name`
     duplicates episode tracking that `_car_api_stale_since` already
     provides: it is set on episode entry, survives the per-cycle
     `_car_api_stale = raw_stale` reset (car.py:705 touches only the flag),
     and is cleared exactly on genuine episode exit (`_exit_stale_mode`,
     car.py:825).
- Proposed fix — rework `check_charger_assignment_contradiction` to this
  shape (exact code is the implementer's choice; behavior is normative):

  ```python
  if self.car_stale_mode_override == CAR_STALE_MODE_FORCE_NOT_STALE:
      return

  raw_home = self._get_raw_is_car_home(time)
  raw_plugged = self._get_raw_is_car_plugged(time)
  has_contradiction = (raw_home is not None and raw_home is False) or (
      raw_plugged is not None and raw_plugged is False
  )
  if not has_contradiction:
      return

  # Manual upgrade: the user's word is ground truth — apply the inferred
  # flags even when the car is already stale from a prior auto episode,
  # but never re-log/re-notify an ongoing episode.
  if manual and not self._car_api_inferred_home:
      self._car_api_inferred_home = True
      self._car_api_inferred_plugged = True

  already_in_episode = self._car_api_stale_since is not None

  if self._car_api_stale or self.car_api_stale_percent_mode:
      return  # already flagged this cycle — flags possibly upgraded above

  self._car_api_stale = True
  self._was_car_api_stale = True
  if already_in_episode:
      return  # re-assert silently across cycles — no log/notification spam

  self._car_api_stale_since = time
  _LOGGER.info(... manual/auto wording, unchanged literals ...)
  notification_body = ...  # unchanged manual/auto bodies
  if self.can_use_charge_percent_constraints():
      self._enter_stale_percent_mode(time)
  self._schedule_person_notification(f"Warning: {self.name}", notification_body, error=True)
  ```

  Normative behavior:
  - **Spam fix preserved** (fix-#02 must-fix #1 must keep passing): for a
    percent-incapable car (e.g. invited), cycle 2's raw refresh resets
    `_car_api_stale`; the check re-asserts it via the
    `already_in_episode` branch without logging or notifying.
    `test_auto_contradiction_no_percent_two_cycles_single_notification`
    (tests/test_car_api_staleness.py:1114) must still pass unchanged in
    spirit (exactly 1 notification, inferred flags stay False on auto).
  - **Manual upgrade**: with an ongoing auto episode (stale, flags False),
    a `manual=True` call sets both inferred flags exactly once,
    idempotently, with NO additional log line and NO additional
    notification. Works both when called directly and via
    `user_set_selected_charger_by_name` (car.py:2596).
  - **One notification per episode**: dedup key is the episode
    (`_car_api_stale_since`), not the charger name. Document in the
    docstring that re-notification is per-episode; a new episode starts
    only after genuine recovery (`_exit_stale_mode`).
  - **No-contradiction path**: plain `return` — no re-arm bookkeeping
    needed (episode exit is owned by the normal staleness recovery paths).
  - **FORCE_NOT_STALE guard unchanged.**
- Deletions:
  - `_contradiction_latched_charger_name` attribute (car.py:246).
  - Its clear in `_exit_stale_mode` (car.py:824).
  - The latch re-arm / latched-branch logic (car.py:894-908) including the
    comment about the auto-mode raw-staleness refresh (round-2 nice-to-have
    #4 — moot).
  - Its clear in `clear_inferred_flags` (car.py:947); revert that method's
    docstring to its pre-latch form.
- Docstring: replace the latch paragraph (car.py:875-878) with the
  per-episode dedup + manual-upgrade description (this also resolves
  round-2 nice-to-have #5 about latch granularity — moot, the latch is
  gone).
- Tests:
  - Update `test_contradiction_skipped_when_already_stale`
    (tests/test_car_api_staleness.py:425): still asserts no notification
    when already stale; split or parametrize so the `manual=True`
    already-stale case now asserts the inferred flags ARE set (upgrade)
    while `manual=False` asserts they are NOT.
  - New test — manual upgrade end-to-end: auto contradiction flags the car
    stale (flags False, 1 notification), then
    `user_set_selected_charger_by_name(<same charger>)`; assert
    `_car_api_inferred_home is True`, `_car_api_inferred_plugged is True`,
    and total notifications == 1.
  - New test — manual upgrade idempotent: second `manual=True` call changes
    nothing and emits nothing.
  - Keep `test_auto_contradiction_no_percent_two_cycles_single_notification`
    green (adjust internals only if the latch attribute leaked in, which
