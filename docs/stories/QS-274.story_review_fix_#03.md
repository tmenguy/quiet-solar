## Findings to fix

### [should-fix] Unify person-forecast string handling; accept the person line in all branches
- File: `custom_components/quiet_solar/ha_model/car.py:1155-1181`
  (`get_car_charge_origin_readable_string`)
- Severity: should-fix
- Source: edge-case-hunter + blind-hunter (root cause), resolved per user decision
- Description: The no-charger branch (`:1158`), the `CAR_CHARGE_TYPE_PERSON_AUTOMATED`
  branch (`:1163`), and the "any other type" fallback (`:1181`) now all return
  `self.get_car_person_readable_forecast_mileage()` directly. This helper is
  None-safe (`car.py:413` → `"No forecasted person"` when no person) and returns
  `"<name>: <forecast>"` otherwise, including `"<name>: No forecast"` when the
  person has no prediction — which the user accepts as the desired display.
- Proposed fix (land the working-tree edit):
  1. Keep the simplified branches as edited (all three delegate to
     `get_car_person_readable_forecast_mileage()`); the `PERSON_NO_FORECAST_STRING`
     import was removed from `car.py` — confirm nothing else in `car.py` references it.
  2. Update the affected assertions in `tests/test_car_charge_origin.py` (lines
     117, 127, 172, 179, 221, 243 previously expected `"No proper Forecast"`) to the
     new behavior:
     - no person attached → `"No forecasted person"`
     - person attached, empty forecast → `"<name>: No forecast"` (e.g. `"Magali: No forecast"`)
     - person attached, populated forecast → unchanged `"<name>: <forecast>"`
     Rename/repurpose `test_empty_forecast_person_origin` so it asserts the
     now-intended `"<name>: No forecast"` line rather than the dropped suppression.
  3. Verify `PERSON_NO_FORECAST_STRING` in `const.py` / `person.py` is still
     referenced where it is produced; if it is now entirely orphaned, remove it
     to avoid dead code (ruff will flag unused imports but not an unused module
     constant).
  4. Doc maintenance: `car.py` is already covered by the PR's `## Doc maintenance`
     section (car-soc-estimation Doc-OK — additive/UI string only); re-confirm
     `check_doc_drift` reconciles after the change.
