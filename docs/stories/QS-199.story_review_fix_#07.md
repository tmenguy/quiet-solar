## Findings to fix

### [nice-to-have] N1 — document why `displayTargetHours` is intentionally raw (COMMENT ONLY)
- File: `custom_components/quiet_solar/ui/resources/shared/qs-ring-duration-base.js` (near line 68, where `const escapedHandleLabel = this._fmt(pctToHours(handlePct));` is computed)
- Severity: nice-to-have
- Source: round-6 blind-hunter (NH1), cleared inert by round-6 edge-case-hunter
- **🚫 DO NOT change logic. This is a comment-only change.**
- Why a logic "fix" is FORBIDDEN: the handle's text label is derived from
  `handlePct` via `this._fmt(pctToHours(handlePct))` — in the shared base
  (line 68) and inline in climate (`:1240`) and water-boiler (`:1126`).
  On the non-default branch `handlePct = hoursToPct(displayTargetHours)` is
  left **unclamped** ON PURPOSE so `pctToHours(handlePct)` round-trips back to
  the TRUE target value. Only the *position* is clamped, by `pctToDeg`
  (`qs-card-base.js:70`). If anyone "fixes" the >100% `handlePct` by clamping
  `handlePct` or `displayTargetHours`, the handle label would render
  `maxHours` instead of the true value → **a real regression across all
  cards.** The current behavior (position pins at the ring top for an
  out-of-range target while the number stays correct) is graceful and
  acceptable.
- Proposed change (comment only): add a short comment at the
  `escapedHandleLabel`/`handlePct` site (and OPTIONALLY a one-line note at each
  card's non-default `displayTargetHours = targetHours` assignment —
  `radiator:213`, `climate:704`, `water-boiler:743`, `on-off-duration:77`)
  stating: *"handlePct is intentionally left unclamped here: the handle label
  round-trips via pctToHours(handlePct) to show the true target; only the
  position is clamped (pctToDeg). Do NOT clamp handlePct/displayTargetHours —
  it would make the label show maxHours instead of the real value."*
- Net effect: prevents a future well-meaning regression; changes no behavior,
  no coverage. (Keep edits to comments only; prefer the single shared-base
  comment to minimize churn.)
