# QS-173 Review Fix #01

Phase 4 adversarial review of PR #174 surfaced an ordering bug in
`scripts/worktree-setup.sh` plus two story-doc nits. This fix lands those
corrections without expanding scope.

## Findings to address

### must-fix

**MF-1 (ordering bug, edge-case-hunter):** the `git push -u origin "$BRANCH"`
call at `scripts/worktree-setup.sh:67` runs **before** the HEAD-verification
block at lines 117-130. If `git worktree add` produced an unexpected HEAD
(the exact scenario the verification exists to detect and recover from),
wrong content is published to `origin/QS_<N>` before the recovery checkout
fires. The push must happen only after HEAD is verified to be on `$BRANCH`.

**MF-2 (partial-state trap on non-fast-forward push, edge-case-hunter):**
if `origin/QS_<N>` already exists with diverged commits, `push -u` fails
under `set -e`. The worktree directory was already created by line 53/56,
so the next invocation hits the `[ -d "$WORKTREE_DIR" ] && exit 1` guard
at line 34 and refuses to retry. AC-3 ("wrong upstream is corrected on
re-setup") cannot fire because the script bails first.

**MF-3 (partial-state trap on any push failure, edge-case-hunter):** same
trap as MF-2, broader trigger set — origin not configured, credential
prompt hang, network failure, pre-push hook reject. Worktree exists, no
symlinks, no HEAD verification, blocked from re-run.

### should-fix

**SF-1 (stale Task 3 references, blind-hunter + coderabbit):**
`docs/stories/QS-173.story.md` Tasks list jumps 1 → 2 → 4, but lines
248, 250, 260, and 274 (plus a few mentions in Adversarial Review Notes)
reference a "Task 3" that does not exist in the document.

**SF-2 (markdownlint MD040/MD046, coderabbit):** fenced code blocks at
lines 48, 96, and 114 of `docs/stories/QS-173.story.md` are missing a
language tag (MD040) or use a mixed code-block style (MD046).

## Out of scope (explicitly NOT in this fix)

## Plan

### Task A — Reorder push and harden partial-state recovery (`scripts/worktree-setup.sh`)

1. **Move** the lines 59-67 block (`# Publish the branch …` comment + the
   `git -C "$WORKTREE_DIR" push -u origin "$BRANCH"` call) from its
   current position (right after worktree creation) to **after** the
   HEAD verification block (after line 130, before the final `echo`
   summary). This fixes MF-1.

2. **Relax** the line 34 directory-exists guard so partial-state recovery
   is possible. Replace the unconditional `exit 1` with a check that
   distinguishes:
   - **Fully set up** (directory exists, is a tracked git worktree on
     the expected branch with upstream `origin/$BRANCH`): print "already
     set up" and exit 0.
   - **Partially set up** (directory exists but worktree HEAD is wrong,
     upstream missing, or directory is not a tracked worktree):
     `git worktree remove --force "$WORKTREE_DIR"` if it's a tracked
     worktree, else `rm -rf "$WORKTREE_DIR"`, then continue with normal
     setup.

   This fixes MF-2 (re-run after non-fast-forward push completes the push
   step on retry) and MF-3 (re-run after any push failure cleans up and
   retries).

3. **Style fix** at line 65 (now in its new position): use `${BRANCH}` 
   consistently rather than mixing `$BRANCH` and `${BRANCH}` in the
   `echo` line — blind-hunter nice-to-have, free to include here.

### Task B — Story doc cleanup (`docs/stories/QS-173.story.md`)

1. **SF-1:** at lines 248, 250, 260, and 274 (plus Adversarial Review
   Notes), replace "Task 3" with the correct task number (likely "Task 4"
   for the post-setup verification reference; check each occurrence in
   context). Do **not** invent a new Task 3.

2. **SF-2:** add a language tag (`bash`, `text`, or `json` as
   appropriate) to the fenced code blocks at lines 48, 96, 114 to clear
   MD040. If MD046 is also flagged, harmonize the fence style across the
   document (use ```` ``` ```` everywhere — no indented code blocks).

### Task C — Manual verification

Re-run the full setup flow against issue 173 (or a throwaway issue
number) on a clean checkout to confirm:

1. First run creates worktree, pushes, sets upstream correctly.
2. Second run with directory present + correct upstream → prints
   "already set up", exits 0.
3. Second run with directory present but upstream wrong (simulate by
   `git branch --unset-upstream` in the worktree) → recovers and
   re-sets upstream.
4. First run with `origin/QS_<N>` ahead of local (simulate by pushing a
   throwaway commit to origin then resetting locally) → either recovers
   on retry or surfaces a clear error message and leaves no partial
   state.
5. Confirm post-fix markdownlint passes on the story file.

No automated tests (per the parent story's "no tests for declarative
config + shell glue" decision).

## Acceptance criteria

- **AC-1:** in `scripts/worktree-setup.sh`, the `git push -u origin

- **AC-2:** re-running `bash scripts/worktree-setup.sh <N>` when the
  worktree directory exists no longer unconditionally exits 1 — it
  either confirms "already set up" and exits 0, or recovers partial
  state and continues.
- **AC-3:** `docs/stories/QS-173.story.md` contains no references to a
  non-existent Task 3, and markdownlint reports no MD040/MD046
  violations on the file.
- **AC-4:** existing ACs from the parent story (AC-1/2/3) still hold —
  upstream is `origin/QS_<N>` after a successful run, including the
  "wrong upstream is corrected on re-setup" case.
