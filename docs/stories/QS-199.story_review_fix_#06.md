## Findings to fix

### [must-fix] G1 — flaky teardown: qs_tag iterator exhausts under full-suite ordering
- File: `tests/test_ui_dashboard.py:875` (`test_generate_qs_tag_high_resolution_distinct_within_one_second`)
- Severity: must-fix (quality gate `pytest` errors=1)
- Source: round-5 acceptance-auditor, confirmed by orchestrator gate re-run
- Scope: TEST-ONLY; test-isolation defect introduced by round-2 S14
- Description: The test does `seq = iter(range(1000, 1050))` (exactly 50 values)
  and `monkeypatch.setattr(time, "time_ns", lambda: next(seq))`, then calls
  `_generate_qs_tag()` exactly 50 times — zero headroom. The global `time.time_ns`
  patch leaks into a pending teardown coroutine scheduled by an earlier test
  (full-suite ordering only), which calls `time_ns` a 51st time → `StopIteration`
  inside a coroutine → `RuntimeError` (PEP 479). Reproduces deterministically in
  the full gate (errors=1); does NOT reproduce in isolation, which is why
  targeted-test verification missed it.
- Proposed fix (minimal): make the patched clock inexhaustible. Replace
  `seq = iter(range(1000, 1050))` with `seq = itertools.count(1000)` (add
  `import itertools`). 50 calls still yield 50 distinct increasing values, so
  `assert len(tags) == 50` and the `.isdigit()` assertion are unchanged — but a
  stray 51st call can never raise. (Equivalent acceptable alternative: a range
  with generous headroom, e.g. `range(1000, 1000000)`.)

### [must-fix] G2 — coverage gap: missing-resources-dir branch untested
- File: `custom_components/quiet_solar/ui/dashboard.py:616-618` (the
  `if not await aiofiles.os.path.isdir(_RESOURCES_DIR): _LOGGER.debug(...); return`
  branch in `_async_update_resources_locked`) — **production code is correct; it
  just lacks a covering test.**
- Severity: must-fix (quality gate coverage 99.99% < 100%)
- Source: round-5 acceptance-auditor, confirmed by orchestrator gate re-run
- Scope: TEST-ONLY (add a test; do NOT change production code)
- Description: No test drives `isdir(_RESOURCES_DIR) == False`. The existing
  `async_update_resources` tests (~`test_ui_dashboard.py:1938-1985`) rely on the
  real bundled resources dir existing (isdir True), and the lock test patches
  isdir to True. So lines 617-618 are never executed → 99.99% coverage → gate
  fails the 100% requirement.
- Proposed fix (minimal): add ONE async test in `tests/test_ui_dashboard.py`
  that patches the missing-dir branch true:
  - patch `dashboard._RESOURCES_DIR` (or `dashboard.aiofiles.os.path.isdir`) so
    `isdir(_RESOURCES_DIR)` returns `False`,
  - stub `aiofiles.os.makedirs` and `_async_remove_orphan_temps` as needed so the
    function reaches the isdir check cleanly,
  - call `await async_update_resources(hass)`,
  - assert it returns early WITHOUT calling `_async_copy_and_register_resources`
    (e.g. patch it with a Mock and assert `not called`), and optionally assert the
    "No bundled resources directory found, skipping" debug log.
  Follow the patching/fixture style of the neighbouring `async_update_resources`
  tests for consistency.
