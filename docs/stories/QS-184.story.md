# QS-184: Migrate OpenCode to the static-agent pipeline; freeze legacy under `legacy/`

GitHub issue: #184
Branch: `QS_184`

## Context

Quiet Solar runs three production AI harnesses (Claude Code, Cursor,
OpenCode) plus a Codex stub. Claude Code and Cursor already use the
**static-agent pipeline**: one checked-in `<harness>/agents/qs-<phase>.md`
file per phase, agents discover task context at runtime via
`python scripts/qs/context.py`, and inter-phase handoff is mediated by
`python scripts/qs/next_step.py` (which dispatches to a
harness-specific launcher in `scripts/qs/launchers/`).

OpenCode is still on the **legacy per-task-rendering pipeline**:

- `_qsprocess_opencode/agent_templates/qs-*.md.tmpl` — 13 Mustache-style
  templates rendered into `<worktree>/.opencode/agents/qs-<phase>-QS-<N>.md`
  by `scripts/qs_opencode/render_agent.py` at each phase boundary.
- `scripts/qs_opencode/` — bespoke `render_agent.py`, `next_step.py`,
  `cleanup_agents.py`, `spawn_session.py`, `launch_opencode.py`,
  `utils.py`, plus their tests under `_qsprocess_opencode/tests/`.
- `.opencode/agents/qs-setup-task.md` and `qs-release.md` — the only
  two static OpenCode agents; both still document the legacy flow
  (e.g. "render qs-create-plan-QS-<N>.md into the worktree").
- `.opencode/commands/setup-task.md` and `release.md` — Claude-style
  slash-command shims that delegate to those two static agents.
- `AGENTS.md` — OpenCode's entry-point document, still describing the
  legacy "two static + per-task rendered" pipeline.
- `docs/opencode-workflow-guide.md` — the full walkthrough of the
  legacy flow.

The launcher infrastructure to drive OpenCode against the static-agent
pipeline already exists: `scripts/qs/launchers/opencode.py` was
delivered in QS-177 and the `launchers/opencode.py:build_payload`
function under `caller="next_step"` POSTs to the OpenCode HTTP API via
`scripts/qs/spawn_session.py` to create a new in-band session with the
next phase's agent bound and a kickoff prompt sent. Per QS-177 AC #12,
that activation API succeeds only when the target
`.opencode/agents/qs-<phase>.md` file exists at OpenCode server startup;
without those files, the session lands on OpenCode's default agent and
phase activation is a silent no-op.

This story closes that gap. After it lands:

- `.opencode/agents/` ships **15 static agent files** (7 orchestrators +
  4 plan reviewer sub-agents + 4 review reviewer sub-agents), each
  body a near-mirror of `.claude/agents/qs-<phase>.md` with OpenCode
  frontmatter conversion.
- `AGENTS.md` is rewritten to mirror `CLAUDE.md`'s structure and point
  exclusively at the new static-agent pipeline.
- `opencode.json` drops its reference to `_qsprocess_opencode/README.md`.
- All legacy machinery (`_qsprocess_opencode/`, `scripts/qs_opencode/`,
  `docs/opencode-workflow-guide.md`, `.opencode/commands/`) moves to a
  top-level `legacy/` directory — **frozen historical code**, never
  modified again, deletion deferred until the new pipeline is proven
  in production.
- `.opencode/commands/` is gone — OpenCode picks the agent via its UI
  agent picker, no slash commands needed.
- Every active reference to `_qsprocess_opencode/`, `scripts/qs_opencode/`,
  or `render_agent.py` is scrubbed from active code, agent definitions,
  and workflow docs. Story files under `docs/stories/` are exempt
  (historical record per user direction).

### Decisions baked into this plan

| Decision | Source |
| --- | --- |
| **Option B: move legacy to `legacy/` subdir**, not hard-delete (preserves rename history; deletion deferred until OpenCode users confirm the new pipeline works) | User direction |
| **No `legacy/README.md`** — directory name is self-documenting; user said "don't care" about ceremony | User direction + scope-guardian I4 |
| **Rewrite the existing `.opencode/agents/qs-setup-task.md` and `qs-release.md` in place** (filenames match `.claude/agents/`; no renames) | User direction |
| **No new slash-command files in `.opencode/commands/`** — agent activation is via OpenCode UI picker | User direction |
| **No PR-time smoke test** — user will exercise in OpenCode UI after merge | User direction |
| **All agents get easy read-only git + gh access** by default; role-bound write tier added only to agents that need it | User direction + critic C1 + dev-proxy C1 |
| **Reuse `_qsprocess_opencode/agent_templates/*.md.tmpl` bash allowlists verbatim per agent** — those already encode the right per-role boundaries | Concrete-planner findings |

from __future__ import annotations

import re
from pathlib import Path

import pytest
import yaml

REPO_ROOT = Path(__file__).resolve().parents[3]
OPENCODE_AGENTS_DIR = REPO_ROOT / ".opencode" / "agents"

SUB_AGENT_NAMES: tuple[str, ...] = (
    "qs-plan-critic",
    "qs-plan-concrete-planner",

# Pattern that catches every "active reference to legacy machinery"
# the migration scrubs. Anchored on three distinct tokens so a future
# rename that affects only one of them still trips the test.
_LEGACY_REFERENCE_RE = re.compile(
    r"_qsprocess_opencode|scripts/qs_opencode|render_agent\.py",
)


def _all_static_agent_names() -> tuple[str, ...]:
    """Phase orchestrators (7) + sub-agents (8) = 15 static agents."""
    from launchers.phases import PHASE_TO_AGENT  # type: ignore[import-not-found]
    return tuple(PHASE_TO_AGENT.values()) + SUB_AGENT_NAMES


def _parse_frontmatter(path: Path) -> tuple[dict, str]:
    """Split an agent file into (frontmatter_dict, body_text).

    Raises ``AssertionError`` on a malformed file — the parametrized
    happy-path tests are the only callers, and any malformed file is
    a test failure by construction.
    """
    text = path.read_text()
    assert text.startswith("---\n"), f"{path} missing opening ``---``"
    rest = text[len("---\n"):]
    end = rest.index("\n---\n")
    return yaml.safe_load(rest[:end]), rest[end + len("\n---\n"):]


@pytest.mark.parametrize("name", _all_static_agent_names())
def test_static_opencode_agent_files_present_and_valid(name: str) -> None:
    path = OPENCODE_AGENTS_DIR / f"{name}.md"
    assert path.is_file(), f"Missing static agent file: {path}"

    fm, _ = _parse_frontmatter(path)
    assert isinstance(fm, dict), f"{path} frontmatter is not a mapping"
    for key in ("description", "mode", "color", "permission"):
        assert key in fm, f"{path} frontmatter missing {key!r}"

    if name in SUB_AGENT_NAMES:
        assert fm["mode"] == "subagent", f"{path} should be mode: subagent"
        assert fm.get("hidden") is True, f"{path} should be hidden: true"
    else:
        assert fm["mode"] == "primary", f"{path} should be mode: primary"


@pytest.mark.parametrize("name", _all_static_agent_names())
def test_no_legacy_path_references_in_opencode_agent_bodies(name: str) -> None:
    path = OPENCODE_AGENTS_DIR / f"{name}.md"
    _, body = _parse_frontmatter(path)
    match = _LEGACY_REFERENCE_RE.search(body)
    assert match is None, (
        f"{path} body still references the legacy pipeline at {match!r}; "
        "scrub the reference or move the file to legacy/."
    )


def test_dev_only_patterns_contains_legacy_not_qsprocess() -> None:
